# 简历优化建议（2026-03-16）

## 总体结论

这份简历基础不错，方向清晰（`C++ + Vulkan + 物理仿真`），但当前核心问题是：

1. 很多内容是“做了什么”，不是“做到什么效果”。
2. 项目描述缺少量化指标（性能、规模、精度、耗时）。
3. 技术栈写得偏泛，和岗位关键词不够强绑定。
4. 有些表述偏学术/抽象，招聘方读不出工程价值。

---

## 先改这 5 个（最高优先级）

1. 加“求职方向/个人摘要”
- 现在开头只有基础信息，没有定位。
- 建议加 2 行：
  - 求职方向：`图形渲染工程师（Vulkan/C++）` 或 `物理仿真工程师`
  - 个人摘要：`熟悉 Vulkan 渲染管线与 GPU 并行计算，具备软体仿真与可视化项目经验，关注性能优化与工程落地。`

2. 项目每条都改成“动作 + 技术 + 结果”
- 现在多是“实现了某算法”，建议补“提升了什么”。
- 模板：`基于[技术]实现[功能]，将[指标]从A提升到B（+X%）`

3. 技术栈拆分成层级
- 不要只写“熟悉 C++/Python/Java”。
- 建议改成：
  - 语言：`C++(主力，C++17/20)、Python(工具链)`
  - 图形：`Vulkan、GLSL、Compute Shader、渲染管线`
  - 数学与仿真：`XPBD、线性代数、数值积分`
  - 工程工具：`CMake、Git、RenderDoc、Nsight（如有）`

4. 教育经历里课程分数精简
- 当前课程+分数太密，HR不容易抓重点。
- 建议只保留 3-5 门最相关课程，或改为：
  - `核心课程：计算机图形学、计算机组成、算法设计与分析、机器学习（均分xx）`

5. 实习经历补“业务背景 + 贡献结果”
- 目前“处理 bug”太泛。
- 建议补：
  - 项目背景（什么系统/模块）
  - 你负责哪块
  - 结果（修复数量、稳定性、上线效果）
