# 拓扑可变子空间物理项目：完整理解知识点清单

## 1. 项目定位与技术主线

项目主题：**拓扑可变的软件子空间物理高效求解研究（C++）**

从你给出的描述看，核心主线是四件事：

1. 基于 **Vulkan Compute Shader** 做 GPU 并行物理求解。
2. 实现 **子空间 + 全空间** 的混合求解框架。
3. 实现 **预计算模板 + 快速插值**，为目标模型快速获得子空间表示。
4. 通过 **凸分解（分块模板化）** 支撑复杂模型与拓扑变化场景。

如果要说“完全理解该项目”，你需要同时掌握：理论推导、数值稳定性、GPU 实现、工程架构、性能证据与局限性分析。

---

## 2. 什么叫“完全理解”（专家验收标准）

你达到“完全理解”至少要能做到：

1. 从连续体力学推导到离散方程，并解释每一步假设。
2. 说明子空间方法在何种情况下有效、何时必须回退到全空间。
3. 讲清混合求解在一个时间步中的完整数据流与耦合策略。
4. 独立实现关键模块（C++ + Vulkan），并解释并行划分合理性。
5. 用实验数据论证速度-精度收益，而不是只给定性描述。
6. 识别并处理失败场景：强非线性、接触冲击、拓扑突变、求解不收敛。
7. 能清楚讲出方法边界与替代方案对比。

## 3. 必备知识点全景图

## 3.1 数学与物理基础

必须掌握：

1. 线性代数：特征分解、SVD、条件数、稀疏矩阵运算。
2. 数值优化：梯度、Hessian、线搜索、约束优化基本思想。
3. 常微分/偏微分方程在动力学中的意义。
4. 连续体力学：应变、应力、材料本构（线弹性/共旋/Neo-Hookean）。
5. 时间积分：显式、半隐式、隐式；稳定性与 CFL 概念。

你需要能解释：

1. 内力和切线刚度（Jacobian）如何从能量函数推导。
2. 为什么刚性系统常用隐式积分。
3. 不同积分法对能量守恒与数值阻尼的影响。

---

## 3.2 网格与离散化（FEM 基础）

必须掌握：

1. 从表面模型（如 OBJ）到四面体体网格的流程和约束（闭合、流形）。
2. FEM 离散：形函数、单元组装、质量矩阵、边界条件。
3. 网格质量指标及其后果（高长宽比、sliver、倒置单元）。
4. 单位一致性与材料参数标定。

你需要能解释：

1. 网格质量为什么直接影响收敛性和稳定性。
2. 网格分辨率对精度、显存、吞吐的影响。

---

## 3.3 子空间方法（项目核心之一）

必须掌握：

1. 降阶表达：`x ≈ x0 + Uq`。
2. 质量/刚度/力在降阶空间中的投影。
3. 基底构建方法：模态基、模态导数、数据驱动基底（如使用）。
4. 降阶误差来源：基底不完备、强局部非线性、接触失配。

你需要能解释：

1. 从全空间方程推导降阶方程。
2. 子空间快在哪里，精度损失在哪里。
3. 如何定义和度量降阶误差（位移、应变能、接触响应）。

---

## 3.4 全空间非线性求解

必须掌握：

1. 非线性迭代流程：残差、Jacobian、Newton/拟 Newton。
2. 线性子问题求解：CG/PCG 等与预条件思路。
3. 稳定化策略：线搜索、正则化、步长控制。
4. 接触与约束常见处理方式（Penalty/拉格朗日乘子/XPBD 思路）。

你需要能解释：

1. 何时必须从子空间切回全空间。
2. 典型不收敛原因及修复路径。

---

## 3.5 子空间 + 全空间混合耦合

必须掌握：

1. 分区耦合策略（局部全空间 + 全局降阶）。
2. 自由度分裂与界面约束传递。
3. 切换触发条件与防抖机制（避免频繁抖动切换）。
4. 动量/能量一致性与漂移修正。

你需要能解释：

1. 单步求解流程图（输入、计算、耦合、更新）。
2. 两种空间间的数据交互是什么。
3. 如何证明“混合方案确实优于纯方案”。

---

## 3.6 拓扑可变与凸分解

必须掌握：

1. 拓扑变化为什么会破坏固定子空间基底假设。
2. 凸分解算法与分块粒度权衡。
3. 分块模板化与局部重建策略。
4. 拓扑变化后的连通关系更新与增量求解思想。

你需要能解释：

1. 为什么分块能提升鲁棒性与实时性。
2. 何种拓扑变化会让方案失效或性能显著下降。

---

## 3.7 预计算模板与快速插值

必须掌握：

1. Offline/Online 分离思想。
2. 模板参数化与模板检索策略。
3. 插值空间选择（线性插值与非线性流形插值差异）。
4. 外推风险与稳定性保护策略。

你需要能解释：

1. 预计算换来的复杂度收益在哪里。
2. 插值有效域如何定义。
3. 速度提升与存储开销如何平衡。

---

## 3.8 Vulkan Compute 与 GPU 并行

必须掌握：

1. Vulkan Compute 基础：队列、描述符、管线、命令缓冲。
2. 同步机制：barrier、pipeline stage、资源可见性。
3. Kernel 设计：线程组规模、访存合并、共享内存、原子冲突。
4. 物理求解相关并行模式：单元并行、归约、稀疏访问优化。
5. CPU-GPU 异步与数据回传最小化。

你需要能解释：

1. 为什么你的 kernel 划分适配当前硬件。
2. 性能瓶颈在哪（算力/带宽/同步/分支发散）。
3. GPU 浮点非确定性对结果复现的影响。

---

## 3.9 C++ 工程架构与可维护性

必须掌握：

1. 现代 C++：RAII、移动语义、资源生命周期管理。
2. 数据导向设计：AoS/SoA、缓存友好、内存对齐。
3. 模块化边界：网格模块、物理模块、求解模块、渲染模块、离线预处理模块。
4. 工程保障：日志、断言、单元测试、集成测试、回归基线。

你需要能解释：

1. 资源所有权和释放路径是否清晰。
2. 如何避免隐藏同步导致的性能“假瓶颈”。

## 3.10 实验设计、评测与论文化表达

必须掌握：

1. 基线对比：纯全空间、纯子空间、你的混合方案。
2. 指标体系：帧耗时、吞吐、误差范数、收敛率、稳定帧占比。

你需要能解释：

1. 速度提升来自哪里。
2. 精度损失来自哪里。
3. 方法边界和下一步优化方向。

## 5. 建议学习与实现顺序

1. 先做稳定的全空间四面体 FEM 基线（CPU 版本）。
2. 接入子空间降阶并与基线对齐验证。
3. 加入混合耦合与切换策略。
4. 加入凸分解与分块模板化。
5. 加入模板插值在线重建。
6. 重核迁移到 Vulkan Compute 并做性能优化。
7. 完成系统化对比实验与消融，形成可答辩证据链。
