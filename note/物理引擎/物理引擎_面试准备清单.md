# 物理引擎：面试准备清单（腾讯 / 网易 / 米哈游方向）

> 目标：从“知道概念”升级到“能讲清原理 + 能落地实现 + 能解释工程取舍”。

## 一、总览结论
- 大厂物理面试通常同时考：算法正确性、稳定性、性能、工程实现、网络同步。
- 仅会背公式不够，要能解释“为什么这样设计”，并能给出调试与优化手段。
- 建议至少具备一个可演示的最小物理系统（刚体 + 碰撞 + 约束求解）的实现经验。

## 二、必须掌握的知识地图（按优先级）

### 1. 数学与力学基础（必会）
- 线性代数：向量、矩阵、四元数、坐标系变换。
- 刚体力学：线速度、角速度、角动量、惯性张量。
- 数值积分：Explicit Euler、Semi-Implicit Euler、Verlet。
- 稳定性：固定时间步、误差累积、能量漂移。

要达到的深度：
- 能解释为什么物理更新常用固定步长（Fixed Timestep）。
- 能解释 Semi-Implicit Euler 相对普通 Euler 的稳定性优势。
- 能写出刚体位姿更新的伪代码。

### 2. 刚体系统（必会）
- 刚体状态：位置、姿态、线速度、角速度、质量、惯量。
- 力与冲量：持续作用 vs 瞬时响应。
- 对象类型：Static / Dynamic / Kinematic。
- 睡眠与唤醒机制（sleep/wake）。

要达到的深度：
- 能解释为什么角色控制常用 kinematic + sweep。
- 能解释惯性张量在局部/世界空间的关系。
- 能定位质量比悬殊引起的不稳定。

### 3. 碰撞检测（必会）
- Broad Phase：Sweep and Prune、动态 AABB 树、BVH、网格分桶。
- Narrow Phase：AABB/OBB/Sphere/Capsule/Convex。
- 关键算法：SAT、GJK、EPA（至少知道各自作用）。
- 接触流形（contact manifold）。
- 连续碰撞检测 CCD：TOI、sweep，防高速穿透。

要达到的深度：
- 能解释为什么要 Broad/Narrow 两层。
- 能解释离散碰撞穿透成因与 CCD 补救机制。
- 能说明 Capsule 在角色系统里的优势。

### 4. 碰撞响应与约束求解（必会）
- 法向冲量、反弹系数 restitution、库仑摩擦。
- 约束类型：距离、铰链、点到点等。
- 迭代求解：Sequential Impulse / PGS 思想。
- 稳定性技巧：Baumgarte、warm start、clamp、split impulse。
- 岛屿拆分（island）与批处理。

要达到的深度：
- 能解释为何常用迭代近似而非一次精确求解。
- 能给出“抖动/爆炸/穿透”排查方向。
- 能写最小接触+摩擦求解框架。

### 5. 物理引擎工程化（必会）
- 固定时间步 + 渲染插值。
- 多线程与同步：物理线程、主线程、任务图。
- 数据布局：SoA、内存池、缓存友好。
- 调试工具：碰撞体可视化、接触点法线、回放与剖析。
- 可复现：随机种子、deterministic 构建、日志回放。

要达到的深度：
- 能解释 Update 与 FixedUpdate 分离必要性。
- 能描述如何复现并定位偶现穿透。
- 能给出移动端预算控制策略。

### 6. 网络同步与确定性（高频）
- Client Prediction、Server Authoritative、Reconciliation。
- Rollback 与输入重演。
- 浮点不确定性来源与跨平台差异。
- 状态压缩与快照同步。

要达到的深度：
- 能解释多人对战里“物理一致性难点”。
- 能给出角色移动预测/回滚流程。
- 能比较强确定性与“近似一致+校正”的工程取舍。

### 7. 动画/渲染联动（常问）
- 动画驱动物理 vs 物理驱动动画。
- Ragdoll、IK 与碰撞同步。
- 布料/绳子/软体：PBD / XPBD 常见方案。
- Character Controller 与普通刚体边界。

要达到的深度：
- 能解释坡面、台阶、滑墙等角色控制细节。
- 能解释布料为什么常用 PBD。

## 三、能力分层（面向大厂）

### L1：及格线
- 能实现 2D 最小物理系统（碰撞 + 冲量 + 摩擦 + 固定步长）。
- 懂 3D 常见形体与算法作用（GJK/EPA/CCD）。
- 能读懂 Box2D/PhysX 的基础数据流。

### L2：强竞争力
- 有可演示 3D mini physics demo（刚体 + 碰撞 + 约束）。
- 有性能优化数据（CPU 时间、碰撞对数、内存占用）。
- 有基本网络同步 demo（预测 + 校正）。

### L3：加分项
- PBD/XPBD 布料或绳子模块。
- 复杂约束稳定性调参经验。
- 大场景 streaming 下的物理分区与激活策略。

## 四、高频面试题清单
- 固定时间步为什么比可变时间步更稳定？
- 为什么会穿透？CCD 如何解决？
- Broad phase 用什么结构，如何选？
- GJK 和 EPA 分别解决什么问题？
- 摩擦约束为什么要切线方向求解？
- 迭代次数如何影响稳定性与性能？
- 质量比差异大时如何稳定？
- 多人对战里物理如何同步？
- 如何定位偶现穿透/抖动问题？
- 如何在移动端做物理性能预算？

## 五、建议项目（最实用）

### 项目1：MiniPhysics3D
目标：
- 刚体 + 碰撞检测 + 冲量求解 + 基础约束 + 可视化调试。

验收：
- 1000+ 动态体稳定运行。
- 有 CCD 开关对比穿透效果。
- 有 profile 数据与优化说明。

### 项目2：Character + NetSync Demo
目标：
- 胶囊角色控制（坡面/台阶/滑墙）+ 客户端预测 + 服务端校正。

验收：
- 高延迟条件下仍可玩。
- 有状态回放与误差校正日志。

## 六、建议学习路线（4周框架）
- 第1周：数学/刚体/积分 + 2D 最小引擎。
- 第2周：Broad/Narrow + 冲量求解 + 稳定性调参。
- 第3周：3D 形体、GJK/EPA、CCD、Character Controller。
- 第4周：网络同步、性能优化、可视化调试与项目包装。

## 七、面试表达模板
> 我把游戏物理引擎理解成三层：
> 第一层是刚体状态更新（积分、时间步、稳定性）；
> 第二层是碰撞检测与约束求解（Broad/Narrow、冲量、摩擦）；
> 第三层是工程化落地（多线程、性能预算、网络同步、可视化调试）。
> 我会重点说明每层的核心算法、瓶颈位置以及工程取舍。
