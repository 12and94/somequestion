# 物理引擎 7：项目实战与面试策略

## 1. 你要准备的两个项目
### 项目 A：MiniPhysics3D
- 内容：刚体、碰撞检测、冲量求解、摩擦、基础约束。
- 加分点：接触点可视化、参数热更新、性能统计面板。

### 项目 B：Character + NetSync Demo
- 内容：角色移动（坡面/台阶）+ 客户端预测 + 服务端校正。
- 加分点：回滚重演、网络条件模拟（延迟/抖动/丢包）。

## 2. 项目验收标准（建议写在简历里）
- 稳定性：长时间运行无明显抖动/穿透。
- 性能：给出机器、场景规模、帧耗时数据。
- 工具链：提供可视化调试和日志回放。

## 3. 面试表达结构（建议固定模板）
- 先说目标：问题是什么。
- 再说方案：用了哪些算法和数据结构。
- 再说取舍：精度、性能、复杂度怎么平衡。
- 再说结果：指标改善了多少。
- 最后说风险：在哪些场景仍有边界问题。

## 4. 高频追问应对
- “为什么不用 XXX 算法？” -> 说明输入规模、平台预算、维护成本。
- “如果出现偶现穿透？” -> 说明复现、日志、可视化、回归流程。
- “如何迁移到移动端？” -> 说明降级策略与预算守门规则。

## 5. 简历落点建议
- 不写“了解物理引擎”。
- 写“实现了什么 + 规模 + 指标 + 问题与修复”。
- 举例：
  - “实现 GJK+EPA 窄相，10k 候选对下窄相耗时下降 28%。”
  - “引入 warm start + split impulse，堆叠场景抖动显著降低。”
