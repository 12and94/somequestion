## 题目：合并K个有序链表

### 标准准确的说法
- 核心思路有两类：
1. 最小堆（priority_queue）维护 K 个链表当前头，弹出最小值后推进对应链表。
2. 分治两两合并，复杂度与归并排序类似。
- 复杂度：
- 最小堆：`O(N log K)`，`N` 是总节点数。
- 分治：`O(N log K)`。
- 边界：空链表、K=0、节点值重复、输入指针为空。

### 通俗易懂的理解
- K 个有序链表像 K 条排好序的流水线，每次只拿每条线最前面的货，选最小的放入结果。

### 面试回答简版模板
`合并 K 个有序链表常用最小堆，维护每条链表当前头节点，整体复杂度 O(N log K)。也可以分治两两合并，复杂度同级。`

### 对应示例
```cpp
struct ListNode {
    int val;
    ListNode* next;
    ListNode(int v) : val(v), next(nullptr) {}
};

ListNode* mergeKLists(std::vector<ListNode*>& lists) {
    auto cmp = [](ListNode* a, ListNode* b) { return a->val > b->val; };
    std::priority_queue<ListNode*, std::vector<ListNode*>, decltype(cmp)> pq(cmp);
    for (auto* h : lists) if (h) pq.push(h);

    ListNode dummy(0);
    ListNode* tail = &dummy;
    while (!pq.empty()) {
        auto* cur = pq.top(); pq.pop();
        tail->next = cur;
        tail = tail->next;
        if (cur->next) pq.push(cur->next);
    }
    tail->next = nullptr;
    return dummy.next;
}
```

## 题目：反转链表 / K个一组反转链表

### 标准准确的说法
- 反转链表本质是指针重连：`prev, cur, next` 三指针迭代。
- K 个一组反转：
- 先判断剩余节点是否 >= K，不足 K 保持原顺序。
- 对每组做局部反转，再连接到前后组。
- 复杂度：
- 反转链表：`O(N)` 时间，`O(1)` 额外空间。
- K 组反转：`O(N)` 时间，`O(1)` 空间。

### 通俗易懂的理解
- 单链表反转就是把箭头全部反向；K 组反转是“每 K 个小段掉头，不够 K 的尾巴不动”。

### 面试回答简版模板
`反转链表是经典三指针重连。K 个一组反转的关键是先探测是否够 K，再做局部反转并拼接，整体 O(N)/O(1)。`

### 对应示例
```cpp
ListNode* reverseList(ListNode* head) {
    ListNode* prev = nullptr;
    ListNode* cur = head;
    while (cur) {
        ListNode* nxt = cur->next;
        cur->next = prev;
        prev = cur;
        cur = nxt;
    }
    return prev;
}
```

```cpp
ListNode* reverseKGroup(ListNode* head, int k) {
    ListNode dummy(0); dummy.next = head;
    ListNode* groupPrev = &dummy;

    while (true) {
        ListNode* kth = groupPrev;
        for (int i = 0; i < k && kth; ++i) kth = kth->next;
        if (!kth) break;

        ListNode* groupNext = kth->next;
        ListNode* prev = groupNext;
        ListNode* cur = groupPrev->next;
        while (cur != groupNext) {
            ListNode* nxt = cur->next;
            cur->next = prev;
            prev = cur;
            cur = nxt;
        }
        ListNode* oldHead = groupPrev->next;
        groupPrev->next = kth;
        groupPrev = oldHead;
    }
    return dummy.next;
}
```

## 题目：删除链表节点（O(1)复杂度）

### 标准准确的说法
- 题目通常给“待删节点指针”，且不是尾节点。
- `O(1)` 删除方式：把后继节点值拷贝到当前节点，再删除后继节点。
- 限制：不能删除尾节点，因为没有后继可拷贝。

### 通俗易懂的理解
- 不是把自己删掉，而是“伪装成后一个节点”，再把后一个节点摘掉。

### 面试回答简版模板
`给定非尾节点时，O(1) 删除是复制 next 节点数据并跳过 next。该方法不适用于尾节点。`

### 对应示例
```cpp
void deleteNodeO1(ListNode* node) {
    if (!node || !node->next) return; // 尾节点无法 O(1) 删除
    ListNode* nxt = node->next;
    node->val = nxt->val;
    node->next = nxt->next;
    delete nxt;
}
```

## 题目：删除链表倒数第N个节点

### 标准准确的说法
- 双指针（快慢）经典题：
- fast 先走 N 步。
- 然后 fast/slow 同步走到 fast 到尾。
- slow 停在待删节点前一个位置。
- 引入哑节点（dummy）统一处理删除头节点情况。
- 复杂度：`O(N)` 时间，`O(1)` 空间。

### 通俗易懂的理解
- 让快指针先“领先 N 格”，最后快到终点时，慢正好站在目标前一格。

### 面试回答简版模板
`用 dummy + 快慢指针，fast 先走 N 步再同步前进，最终 slow 指向待删节点前驱，可一次遍历完成。`

### 对应示例
```cpp
ListNode* removeNthFromEnd(ListNode* head, int n) {
    ListNode dummy(0); dummy.next = head;
    ListNode* fast = &dummy;
    ListNode* slow = &dummy;

    for (int i = 0; i < n; ++i) fast = fast->next;
    while (fast->next) {
        fast = fast->next;
        slow = slow->next;
    }
    ListNode* del = slow->next;
    slow->next = del->next;
    delete del;
    return dummy.next;
}
```

## 题目：环形链表检测

### 标准准确的说法
- Floyd 判圈（快慢指针）：
- slow 每次走 1，fast 每次走 2。
- 若有环必相遇，无环 fast 会先到空。
- 若需找环入口：相遇后一指针回头节点，两指针同速再走，相遇点即入口。
- 复杂度：`O(N)` 时间，`O(1)` 空间。

### 通俗易懂的理解
- 操场跑圈，快的人迟早追上慢的人；追不上说明根本没圈。

### 面试回答简版模板
`环检测用 Floyd 快慢指针，O(N)/O(1)。有环会相遇，无环 fast 先到空；相遇后可进一步求入口。`

### 对应示例
```cpp
bool hasCycle(ListNode* head) {
    ListNode* slow = head;
    ListNode* fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) return true;
    }
    return false;
}

ListNode* detectCycleEntry(ListNode* head) {
    ListNode* slow = head;
    ListNode* fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            ListNode* p = head;
            while (p != slow) {
                p = p->next;
                slow = slow->next;
            }
            return p;
        }
    }
    return nullptr;
}
```

## 题目：实现双向链表

### 标准准确的说法
- 双向链表节点维护 `prev/next`。
- 头尾哑节点设计能统一插入删除边界。
- 基本操作：
- 指定节点后插入、删除指定节点、头插/尾插。
- 每次操作需同时维护前后两个方向指针。

### 通俗易懂的理解
- 单链表像单行道，双向链表像双行道，可以前进也能后退。

### 面试回答简版模板
`实现双向链表建议用 head/tail 哑节点，插删都转成“在某节点后插入”与“删除某节点”，可稳定处理边界。`

### 对应示例
```cpp
struct DNode {
    int val;
    DNode* prev;
    DNode* next;
    DNode(int v=0) : val(v), prev(nullptr), next(nullptr) {}
};

class DoublyList {
public:
    DoublyList() {
        head = new DNode();
        tail = new DNode();
        head->next = tail;
        tail->prev = head;
    }

    void push_front(int x) { insert_after(head, new DNode(x)); }
    void push_back(int x)  { insert_after(tail->prev, new DNode(x)); }

    void erase(DNode* node) {
        if (!node || node == head || node == tail) return;
        node->prev->next = node->next;
        node->next->prev = node->prev;
        delete node;
    }

private:
    DNode* head;
    DNode* tail;

    void insert_after(DNode* pos, DNode* node) {
        node->next = pos->next;
        node->prev = pos;
        pos->next->prev = node;
        pos->next = node;
    }
};
```

## 题目：链表奇偶节点排序

### 标准准确的说法
- 常见题意是“按位置奇偶分组”：把原链表奇数位节点放前，偶数位放后，保持各自相对顺序。
- 方法：维护 odd 指针链、even 指针链，最后拼接。
- 复杂度：`O(N)` 时间，`O(1)` 额外空间。

### 通俗易懂的理解
- 像按排号把“奇数号队伍”和“偶数号队伍”分开排，再把偶数队接到奇数队后面。

### 面试回答简版模板
`链表奇偶重排用两个链并行维护 odd/even，遍历一次完成并在末尾拼接，保持稳定顺序，复杂度 O(N)/O(1)。`

### 对应示例
```cpp
ListNode* oddEvenList(ListNode* head) {
    if (!head || !head->next) return head;
    ListNode* odd = head;
    ListNode* even = head->next;
    ListNode* evenHead = even;

    while (even && even->next) {
        odd->next = even->next;
        odd = odd->next;
        even->next = odd->next;
        even = even->next;
    }
    odd->next = evenHead;
    return head;
}
```
