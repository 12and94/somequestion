## 题目：两数之和 / 三数之和 / 四数之和

### 标准准确的说法
- 两数之和：哈希表记录已访问值，查 `target - x`，`O(N)`。
- 三数之和：排序 + 固定一个数 + 双指针，`O(N^2)`，注意去重。
- 四数之和：排序 + 双层固定 + 双指针，`O(N^3)`，同样要去重。

### 通俗易懂的理解
- 两数之和像“边走边记账”；三数/四数之和像“先排队，再左右夹逼找组合”。

### 面试回答简版模板
`Two Sum 用哈希 O(N)。Three/Four Sum 常用排序+双指针，时间复杂度分别 O(N^2)/O(N^3)，难点在去重和边界处理。`

### 对应示例
```cpp
std::vector<int> twoSum(std::vector<int>& nums, int target) {
    std::unordered_map<int, int> mp;
    for (int i = 0; i < (int)nums.size(); ++i) {
        int need = target - nums[i];
        if (mp.count(need)) return {mp[need], i};
        mp[nums[i]] = i;
    }
    return {};
}
```

```cpp
std::vector<std::vector<int>> threeSum(std::vector<int> nums) {
    std::sort(nums.begin(), nums.end());
    std::vector<std::vector<int>> ans;
    int n = (int)nums.size();
    for (int i = 0; i < n; ++i) {
        if (i && nums[i] == nums[i - 1]) continue;
        int l = i + 1, r = n - 1;
        while (l < r) {
            long long s = 1LL * nums[i] + nums[l] + nums[r];
            if (s < 0) ++l;
            else if (s > 0) --r;
            else {
                ans.push_back({nums[i], nums[l], nums[r]});
                while (l < r && nums[l] == nums[l + 1]) ++l;
                while (l < r && nums[r] == nums[r - 1]) --r;
                ++l; --r;
            }
        }
    }
    return ans;
}
```

## 题目：数组中的重复元素

### 标准准确的说法
- 常见三解：
1. 哈希集合：`O(N)` 时间，`O(N)` 空间。
2. 排序后扫描：`O(N log N)` 时间，`O(1)` 额外空间（原地排序）。
3. 位图/布尔数组：当值域有限时更高效。

### 通俗易懂的理解
- 就是检查“是否有人报到两次”，可以用签到表（哈希）最快查重。

### 面试回答简版模板
`重复元素检测优先哈希集合 O(N)。若空间受限可排序后线性扫描。值域可控时位图更省常数。`

### 对应示例
```cpp
bool containsDuplicate(const std::vector<int>& nums) {
    std::unordered_set<int> st;
    for (int x : nums) {
        if (st.count(x)) return true;
        st.insert(x);
    }
    return false;
}
```

## 题目：接雨水

### 标准准确的说法
- 双指针解法：
- 维护左右边界最大高度 `lmax/rmax`。
- 较低侧决定当前可接水量。
- 复杂度：`O(N)` 时间，`O(1)` 空间。
- 另有单调栈解法，适合延展“柱状图”系列题。

### 通俗易懂的理解
- 每个位置能装多少水，取决于左右挡板较矮那一边。

### 面试回答简版模板
`接雨水最优常用双指针，维护 lmax/rmax，较低侧先结算，可做到 O(N) 时间 O(1) 空间。`

### 对应示例
```cpp
int trap(const std::vector<int>& h) {
    int l = 0, r = (int)h.size() - 1;
    int lmax = 0, rmax = 0, ans = 0;
    while (l < r) {
        if (h[l] < h[r]) {
            lmax = std::max(lmax, h[l]);
            ans += lmax - h[l];
            ++l;
        } else {
            rmax = std::max(rmax, h[r]);
            ans += rmax - h[r];
            --r;
        }
    }
    return ans;
}
```

## 题目：二维数组查找目标值

### 标准准确的说法
- 经典条件：每行升序、每列升序。
- 从右上角（或左下角）开始：
- 当前值 > target：向左。
- 当前值 < target：向下。
- 复杂度：`O(m+n)`。

### 通俗易懂的理解
- 右上角像“分叉路口”：左边都更小，下边都更大，每步都能排除一整行或一整列。

### 面试回答简版模板
`二维有序矩阵查找从右上角起步，比较后左移或下移，每步排除一维，复杂度 O(m+n)。`

### 对应示例
```cpp
bool searchMatrix(const std::vector<std::vector<int>>& a, int target) {
    if (a.empty() || a[0].empty()) return false;
    int m = (int)a.size(), n = (int)a[0].size();
    int i = 0, j = n - 1;
    while (i < m && j >= 0) {
        if (a[i][j] == target) return true;
        if (a[i][j] > target) --j;
        else ++i;
    }
    return false;
}
```

## 题目：搜索旋转排序数组

### 标准准确的说法
- 旋转数组仍可二分。
- 每次判断哪一半有序，再判断 target 是否落在有序半区。
- 复杂度：`O(log N)`（无重复值版本）。

### 通俗易懂的理解
- 虽然整体拧了一下，但每次中点总有一侧是“正常升序段”，可继续二分定位。

### 面试回答简版模板
`旋转数组搜索本质是改造二分：先识别有序半边，再判断目标是否在该区间，时间 O(log N)。`

### 对应示例
```cpp
int searchRotated(const std::vector<int>& nums, int target) {
    int l = 0, r = (int)nums.size() - 1;
    while (l <= r) {
        int m = l + (r - l) / 2;
        if (nums[m] == target) return m;
        if (nums[l] <= nums[m]) { // 左半有序
            if (nums[l] <= target && target < nums[m]) r = m - 1;
            else l = m + 1;
        } else { // 右半有序
            if (nums[m] < target && target <= nums[r]) l = m + 1;
            else r = m - 1;
        }
    }
    return -1;
}
```

## 题目：LRU缓存实现

### 标准准确的说法
- 目标：`get/put` 平均 `O(1)`。
- 标准结构：
- 哈希表：key -> 双向链表节点。
- 双向链表：维护最近使用顺序（头最新，尾最旧）。
- `get`：命中后节点移到头。
- `put`：更新或新插；容量满时淘汰尾节点。

### 通俗易懂的理解
- 哈希负责“秒查地址”，双向链表负责“维护新旧顺序”，两者配合才是 O(1) LRU。

### 面试回答简版模板
`LRU 用 unordered_map + doubly linked list。map 做 O(1) 定位，链表维护访问顺序，尾部淘汰最久未使用元素。`

### 对应示例
```cpp
class LRUCache {
    struct Node {
        int k, v;
        Node* prev;
        Node* next;
        Node(int key=0, int val=0): k(key), v(val), prev(nullptr), next(nullptr) {}
    };

public:
    explicit LRUCache(int cap): cap_(cap) {
        head_ = new Node(); tail_ = new Node();
        head_->next = tail_; tail_->prev = head_;
    }

    int get(int key) {
        auto it = mp_.find(key);
        if (it == mp_.end()) return -1;
        moveToFront(it->second);
        return it->second->v;
    }

    void put(int key, int value) {
        auto it = mp_.find(key);
        if (it != mp_.end()) {
            it->second->v = value;
            moveToFront(it->second);
            return;
        }
        if ((int)mp_.size() == cap_) {
            Node* del = tail_->prev;
            remove(del);
            mp_.erase(del->k);
            delete del;
        }
        Node* node = new Node(key, value);
        addFront(node);
        mp_[key] = node;
    }

private:
    int cap_;
    Node *head_, *tail_;
    std::unordered_map<int, Node*> mp_;

    void remove(Node* x) {
        x->prev->next = x->next;
        x->next->prev = x->prev;
    }

    void addFront(Node* x) {
        x->next = head_->next;
        x->prev = head_;
        head_->next->prev = x;
        head_->next = x;
    }

    void moveToFront(Node* x) {
        remove(x);
        addFront(x);
    }
};
```
