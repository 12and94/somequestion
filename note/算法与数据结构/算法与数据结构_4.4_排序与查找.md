## 题目：快速排序的原理？最坏情况？
>[!NOTE]- 标准准确的说法
>- 快排核心：分治 + partition。
>- 选基准（pivot），把小于 pivot 放左边，大于 pivot 放右边，再递归子区间。
>- 平均复杂度 `O(N log N)`，最坏 `O(N^2)`。
>- 最坏情况常见于：每次 pivot 选到区间极值（例如有序数组+固定端点 pivot）。
>- 优化：随机化 pivot、三数取中、尾递归优化、小区间切插入排序。

>[!tip]- 代码示例
>```cpp
>int partition(std::vector<int>& a, int l, int r) {
>   int pivot = a[r];
>   int i = l;
>   for (int j = l; j < r; ++j) {
>       if (a[j] <= pivot) std::swap(a[i++], a[j]);
>   }
>   std::swap(a[i], a[r]);
>   return i;
>}
>
>void quickSort(std::vector<int>& a, int l, int r) {
>   if (l >= r) return;
>   int p = partition(a, l, r);
>   quickSort(a, l, p - 1);
>   quickSort(a, p + 1, r);
>}
>```

>[!quote]- 通俗易懂的理解
>- 快排像“选一个标尺，把人群按高矮分成两边，再分别继续分组”。

>[!example]- 面试回答简版模板
>`快排通过 partition 把数组分成左右两半递归处理，平均 O(N log N)。最坏 O(N^2) 主要由极端 pivot 触发，可通过随机化或三数取中缓解。`

>[!info]- 对应示例
>```cpp
>int partition(std::vector<int>& a, int l, int r) {
>   int pivot = a[r];
>   int i = l;
>   for (int j = l; j < r; ++j) {

## 题目：各种排序算法的复杂度？
>[!NOTE]- 标准准确的说法
>- 常见复杂度对比（平均时间）：
>- 冒泡/选择/插入：`O(N^2)`。
>- 快速排序：`O(N log N)`，最坏 `O(N^2)`。
>- 归并排序：`O(N log N)`，稳定，额外空间 `O(N)`。
>- 堆排序：`O(N log N)`，原地，通常不稳定。
>- 计数/桶/基数排序：在值域受限或位数固定时可近线性。
>- 稳定性：归并、冒泡、插入稳定；快排、堆排通常不稳定。

>[!tip]- 代码示例
>```text
>算法     平均时间      最坏时间      额外空间    稳定性
>冒泡     O(N^2)        O(N^2)        O(1)        稳定
>插入     O(N^2)        O(N^2)        O(1)        稳定
>选择     O(N^2)        O(N^2)        O(1)        不稳定
>快排     O(NlogN)      O(N^2)        O(logN)     不稳定
>归并     O(NlogN)      O(NlogN)      O(N)        稳定
>堆排     O(NlogN)      O(NlogN)      O(1)        不稳定
>```

>[!quote]- 通俗易懂的理解
>- 排序题核心不是背名字，而是记“时间复杂度 + 空间复杂度 + 稳定性”三元组。

>[!example]- 面试回答简版模板
>`工程上常比较排序的三件事：时间、空间、稳定性。通用比较排序下界是 O(N log N)，归并稳定但耗空间，堆排序原地但不稳定，快排平均快但最坏退化。`

>[!info]- 对应示例
>```text
>算法     平均时间      最坏时间      额外空间    稳定性
>冒泡     O(N^2)        O(N^2)        O(1)        稳定
>插入     O(N^2)        O(N^2)        O(1)        稳定
>选择     O(N^2)        O(N^2)        O(1)        不稳定
>快排     O(NlogN)      O(N^2)        O(logN)     不稳定
>归并     O(NlogN)      O(NlogN)      O(N)        稳定
>堆排     O(NlogN)      O(NlogN)      O(1)        不稳定
>```

## 题目：堆排序原理？建堆和调整？
>[!NOTE]- 标准准确的说法
>- 堆排序步骤：
>1. 建最大堆。
>2. 交换堆顶与末尾，缩小堆区间。
>3. 对堆顶做下滤（heapify），重复直到有序。
>- 建堆：自底向上从最后一个非叶节点开始 heapify，`O(N)`。
>- 总复杂度：`O(N log N)`，原地排序，通常不稳定。

>[!tip]- 代码示例
>```cpp
>void heapify(std::vector<int>& a, int n, int i) {
>   while (true) {
>       int l = 2 * i + 1, r = 2 * i + 2, mx = i;
>       if (l < n && a[l] > a[mx]) mx = l;
>       if (r < n && a[r] > a[mx]) mx = r;
>       if (mx == i) break;
>       std::swap(a[i], a[mx]);
>       i = mx;
>   }
>}
>
>void heapSort(std::vector<int>& a) {
>   int n = (int)a.size();
>   for (int i = n / 2 - 1; i >= 0; --i) heapify(a, n, i); // 建堆
>   for (int end = n - 1; end > 0; --end) {
>       std::swap(a[0], a[end]);
>       heapify(a, end, 0);
>   }
>}
>```

>[!quote]- 通俗易懂的理解
>- 每轮把“冠军”（堆顶最大）放到最终位置，然后重新选下一轮冠军。

>[!example]- 面试回答简版模板
>`堆排序先 O(N) 建最大堆，再反复把堆顶换到末尾并下滤恢复堆，整体 O(N log N)、O(1) 额外空间。`

>[!info]- 对应示例
>```cpp
>void heapify(std::vector<int>& a, int n, int i) {
>   while (true) {
>       int l = 2 * i + 1, r = 2 * i + 2, mx = i;
>       if (l < n && a[l] > a[mx]) mx = l;
>       if (r < n && a[r] > a[mx]) mx = r;
>       if (mx == i) break;
>       std::swap(a[i], a[mx]);
>       i = mx;
>   }
>}
>
>void heapSort(std::vector<int>& a) {
>   int n = (int)a.size();
>   for (int i = n / 2 - 1; i >= 0; --i) heapify(a, n, i); // 建堆
>   for (int end = n - 1; end > 0; --end) {
>       std::swap(a[0], a[end]);
>       heapify(a, end, 0);
>   }
>}
>```

## 题目：归并排序？如何优化空间复杂度？
>[!NOTE]- 标准准确的说法
>- 归并排序：分治把数组拆到单元素，再两两归并有序段。
>- 时间复杂度稳定 `O(N log N)`，稳定排序。
>- 空间优化方向：
>- 复用一份全局临时数组，避免每层递归重复分配。
>- 自底向上迭代版可减少递归开销。
>- 链表归并排序可做到 `O(1)` 额外指针空间（不计递归）。

>[!tip]- 代码示例
>```cpp
>void mergeSort(std::vector<int>& a, int l, int r, std::vector<int>& buf) {
>   if (l >= r) return;
>   int m = l + (r - l) / 2;
>   mergeSort(a, l, m, buf);
>   mergeSort(a, m + 1, r, buf);
>   int i = l, j = m + 1, k = l;
>   while (i <= m && j <= r) buf[k++] = (a[i] <= a[j] ? a[i++] : a[j++]);
>   while (i <= m) buf[k++] = a[i++];
>   while (j <= r) buf[k++] = a[j++];
>   for (int t = l; t <= r; ++t) a[t] = buf[t];
>}
>```

>[!quote]- 通俗易懂的理解
>- 先把大问题拆成小有序块，再像拉拉链一样把小块合并。

>[!example]- 面试回答简版模板
>`归并排序是稳定 O(N log N)。空间优化关键是复用辅助数组或用迭代版减少额外开销；数组场景通常仍需 O(N) 辅助空间。`

>[!info]- 对应示例
>```cpp
>void mergeSort(std::vector<int>& a, int l, int r, std::vector<int>& buf) {
>   if (l >= r) return;
>   int m = l + (r - l) / 2;
>   mergeSort(a, l, m, buf);
>   mergeSort(a, m + 1, r, buf);
>   int i = l, j = m + 1, k = l;
>   while (i <= m && j <= r) buf[k++] = (a[i] <= a[j] ? a[i++] : a[j++]);
>   while (i <= m) buf[k++] = a[i++];
>   while (j <= r) buf[k++] = a[j++];
>   for (int t = l; t <= r; ++t) a[t] = buf[t];
>}
>```

## 题目：二分查找
>[!NOTE]- 标准准确的说法
>- 前提：序列有序。
>- 维护区间 `[l, r]`，中点比较后缩小一半区间。
>- 常见变形：
>- 找第一个 >= target（lower_bound）
>- 找第一个 > target（upper_bound）
>- 边界细节：`mid` 计算防溢出，循环不变量要稳定。

>[!tip]- 代码示例
>```cpp
>int binarySearch(const std::vector<int>& a, int target) {
>   int l = 0, r = (int)a.size() - 1;
>   while (l <= r) {
>       int m = l + (r - l) / 2;
>       if (a[m] == target) return m;
>       if (a[m] < target) l = m + 1;
>       else r = m - 1;
>   }
>   return -1;
>}
>```

>[!quote]- 通俗易懂的理解
>- 每次都把“不可能区间”砍掉一半，像猜数字“太大了往左，太小了往右”。

>[!example]- 面试回答简版模板
>`二分查找本质是维护区间不变量并每轮砍半，标准查找 O(log N)，变形题关键是边界定义和返回条件。`

>[!info]- 对应示例
>```cpp
>int binarySearch(const std::vector<int>& a, int target) {
>   int l = 0, r = (int)a.size() - 1;
>   while (l <= r) {
>       int m = l + (r - l) / 2;
>       if (a[m] == target) return m;
>       if (a[m] < target) l = m + 1;
>       else r = m - 1;
>   }
>   return -1;
>}
>```
>
>```cpp
>int lowerBound(const std::vector<int>& a, int target) {
>   int l = 0, r = (int)a.size(); // [l, r)
>   while (l < r) {
>       int m = l + (r - l) / 2;
>       if (a[m] < target) l = m + 1;
>       else r = m;
>   }
>   return l;
>}
>```

## 题目：快速选择（TopK问题）
>[!NOTE]- 标准准确的说法
>- Quickselect 基于 partition，只递归进入包含第 k 小元素的一侧。
>- 平均复杂度 `O(N)`，最坏 `O(N^2)`（可用随机化降低退化概率）。
>- TopK 常见两类：
>- 离线一次性：quickselect 或部分排序。
>- 在线数据流：大小为 K 的最小堆/最大堆。

>[!tip]- 代码示例
>```cpp
>int quickSelect(std::vector<int>& a, int l, int r, int k) {
>   if (l == r) return a[l];
>   int p = partition(a, l, r);
>   if (p == k) return a[p];
>   if (p > k) return quickSelect(a, l, p - 1, k);
>   return quickSelect(a, p + 1, r, k);
>}
>```

>[!quote]- 通俗易懂的理解
>- 不把全班都排好名次，只把“第 K 名在哪个半区”一路缩小找到即可。

>[!example]- 面试回答简版模板
>`TopK 离线场景可用 Quickselect，平均 O(N)；在线场景常用堆维持 K 个候选，单次更新 O(log K)。`

>[!info]- 对应示例
>```cpp
>int quickSelect(std::vector<int>& a, int l, int r, int k) {
>   if (l == r) return a[l];
>   int p = partition(a, l, r);
>   if (p == k) return a[p];
>   if (p > k) return quickSelect(a, l, p - 1, k);
>   return quickSelect(a, p + 1, r, k);
>}
>```

## 题目：快速幂 / 快速幂求余
>[!NOTE]- 标准准确的说法
>- 利用指数二进制展开：
>- `a^b` 可按 `b` 的每一位拆成平方连乘。
>- 时间复杂度从 `O(b)` 降到 `O(log b)`。
>- 求余版：每次乘法后取模，避免溢出并保持同余性质。

>[!tip]- 代码示例
>```cpp
>long long modPow(long long a, long long b, long long mod) {
>   long long ans = 1 % mod;
>   a %= mod;
>   while (b > 0) {
>       if (b & 1) ans = (__int128)ans * a % mod;
>       a = (__int128)a * a % mod;
>       b >>= 1;
>   }
>   return ans;
>}
>```

>[!quote]- 通俗易懂的理解
>- 指数每次减半，底数每次平方，像“按二进制位选乘法套餐”。

>[!example]- 面试回答简版模板
>`快速幂用二进制拆指数，循环中“看最低位决定乘不乘，底数平方，指数右移”，复杂度 O(log b)。求余版每步取模。`

>[!info]- 对应示例
>```cpp
>long long modPow(long long a, long long b, long long mod) {
>   long long ans = 1 % mod;
>   a %= mod;
>   while (b > 0) {
>       if (b & 1) ans = (__int128)ans * a % mod;
>       a = (__int128)a * a % mod;
>       b >>= 1;
>   }
>   return ans;
>}
>```
