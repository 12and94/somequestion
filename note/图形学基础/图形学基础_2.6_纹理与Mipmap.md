## 题目：Mipmap的原理和作用？如何确定使用哪一层？
>[!NOTE]- 标准准确的说法
>- **先说人话**
>- Mipmap 就是一张图的“多套分辨率版本”：原图、1/2、1/4、1/8 ...
>- 渲染时不是总用原图，而是按当前像素需要的细节量选择合适层级。
>- **为什么一定要有它**
>- 1) 减少远处闪烁和锯齿（高频细节别硬采）。
>- 2) 降低带宽和缓存压力（远处没必要读超大贴图）。
>- 3) 让 LOD 变化更平滑（配合双线性/三线性过滤）。
>- **LOD 怎么来的（不用死记）**
>- GPU 会看 `uv` 在屏幕上的变化速度（导数）。
>- `uv` 变化快，说明一个像素覆盖了更大纹理区域，就应该用更小的 mip 层。
>- 常见表达：
> $$
> \lambda = \log_2\left(\max(\rho_x,\rho_y)\right)
> $$
>- 其中 $\rho_x, \rho_y$ 来自 `ddx/ddy` 估计的 footprint 大小。
>- **实际采样**
>- 整数层：直接取该 mip。
>- 小数层：在相邻两层做线性混合（trilinear），减少层间跳变。

>[!tip]- 代码示例
>```mermaid
>flowchart LR
>  A[像素采样请求] --> B[计算 ddx/ddy]
>  B --> C[估算 footprint]
>  C --> D[得到 LOD]
>  D --> E[选择 mip 层]
>  E --> F[过滤并返回颜色]
>```
>
>```cpp
>// 概念伪代码
>float lod = ComputeLOD(uv_ddx, uv_ddy);
>float3 c = SampleTextureTrilinear(tex, uv, lod);
>```

>[!quote]- 通俗易懂的理解
>- 把贴图想成“同一张照片的原图和缩略图”。
>- 远处物体就像手机相册缩略图，不需要每次都加载原图细节。

>[!example]- 面试回答简版模板
>`Mipmap是纹理的多分辨率链，作用是减少缩小采样走样并降低带宽。LOD通常由uv导数自动估计，采样时常用三线性在相邻mip之间平滑过渡。`

## 题目：法线贴图是怎么存储的？为什么是蓝色？
>[!NOTE]- 标准准确的说法
>- **先说结论**
>- 法线贴图通常存的是“切线空间法线”（不是世界空间法线）。
>- 法线分量原本范围是 `[-1,1]`，纹理只能存 `[0,1]`，所以要映射。
>- **编码/解码**
>- 编码：
> $$
> n_{enc} = n_{ts} \cdot 0.5 + 0.5
> $$
>- 解码：
> $$
> n_{ts} = normalize(n_{enc} \cdot 2.0 - 1.0)
> $$
>- **为什么看起来偏蓝**
>- 大多数表面“默认法线”朝外，切线空间里通常接近 `(0,0,1)`。
>- 编码后接近 `(0.5,0.5,1.0)`，也就是蓝通道高，所以整图偏蓝。
>- **用的时候要注意**
>- 需要用 TBN 把切线空间法线变到世界/观察空间再参与光照。
>- 法线贴图通常按线性空间采样，不走 sRGB。

>[!tip]- 代码示例
>```cpp
>// 解码切线空间法线
>float3 nTS = texNormal.rgb * 2.0 - 1.0;
>nTS = normalize(nTS);
>
>// 变换到世界空间
>float3 nWS = normalize(mul(TBN, nTS));
>```

>[!quote]- 通俗易懂的理解
>- 蓝色不是美术“涂蓝了”，而是数学映射结果：
>- 默认法线朝外（z 大），映射到颜色后蓝通道自然更亮。

>[!example]- 面试回答简版模板
>`法线贴图通常存切线空间法线，先把[-1,1]映射到[0,1]存储，采样后再反映射并归一化。之所以偏蓝，是因为默认法线z分量普遍较大。`

## 题目：各向异性采样是什么？
>[!NOTE]- 标准准确的说法
>- **它在解决什么问题**
>- 斜视角看地面/墙面时，纹理 footprint 会被拉成长条。
>- 普通各向同性采样对这个长条处理不好，会模糊或糊成一片。
>- **各向异性采样的核心**
>- 在“拉伸最严重的方向”增加采样，保住斜视角细节。
>- 常见可调等级：2x / 4x / 8x / 16x。
>- **好处与代价**
>- 好处：道路、砖墙、地板等斜角纹理明显更清晰。
>- 代价：采样次数增加，带宽和算力开销上升。
>- **什么时候开高**
>- 第三人称地面、大范围户外路面、赛车类道路材质。
>- UI/近景平视不一定需要高等级。

>[!tip]- 代码示例
>```cpp
>// API层面通常是采样器配置，不是手写循环
>SamplerDesc s;
>s.filter = ANISOTROPIC;
>s.maxAnisotropy = 8; // 2/4/8/16
>```

>[!quote]- 通俗易懂的理解
>- 你斜着看地砖时，细节会“被拉扁”。
>- 各向异性采样就是沿这个拉扁方向多看几眼，把细节救回来。

>[!example]- 面试回答简版模板
>`各向异性采样用于解决斜视角纹理拉伸导致的模糊问题：它会在主拉伸方向增加采样，提高地面等材质清晰度，代价是更高采样开销。`

## 题目：Mipmap的额外存储空间是多少？
>[!NOTE]- 标准准确的说法
>- **结论先给**
>- 2D 纹理完整 mip 链总像素量约为原图的 `4/3`。
>- 所以“额外开销”约是原图的 `1/3`（约 33%）。
>- **为什么是 33%**
>- 每降一级，面积变为原来的 `1/4`：
> $$
> 1 + \frac{1}{4} + \frac{1}{16} + \cdots = \frac{4}{3}
> $$
>- 因此：
> $$
> extra = \frac{4}{3} - 1 = \frac{1}{3}
> $$
>- **工程上为什么不完全等于 33%**
>- 块压缩格式（如 ASTC/BC）有块对齐；
>- 非 2^n 尺寸会有边界补齐；
>- 有的资源可能只保留到某个 mip 层，不是完整链。

>[!tip]- 代码示例
>```cpp
>// 概念：估算完整 mip 链大小
>size_total ≈ size_base * 4.0 / 3.0;
>size_extra ≈ size_base / 3.0;
>```

>[!quote]- 通俗易懂的理解
>- 原图是“100%”，完整 mip 链大概再加“33%”。
>- 这 33% 换来的是更稳画质和更省带宽，通常很划算。

>[!example]- 面试回答简版模板
>`2D纹理完整Mipmap链总量约是原图4/3，额外开销约33%。实际项目会受压缩块对齐、尺寸和是否完整链影响。`

## 题目：纹理压缩有哪些？移动端的纹理压缩？
>[!NOTE]- 标准准确的说法
>- **先抓主线：这是“平台兼容 + 画质 + 带宽”的平衡题**
>- 常见家族：
>- `BCn/DXT`：桌面平台常见（DirectX 生态成熟）。
>- `ETC2/EAC`：OpenGL ES 通用移动兼容方案。
>- `ASTC`：移动端主流高质量方案，块大小可变（质量/码率可调）。
>- `PVRTC`：部分平台历史常见（老设备兼容场景）。
>- **移动端为什么常推 ASTC**
>- 质量和压缩率平衡好；
>- 支持按内容选块大小（例如 UI 用小块保细节，背景可大块省空间）。
>- **落地建议（简单可执行）**
>- 新项目优先：ASTC（设备支持时）。
>- 兼容保底：ETC2。
>- 导入管线按贴图类型分档（Albedo / Normal / UI 分开压缩策略）。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[目标平台] --> B{支持ASTC?}
>  B -->|是| C[优先ASTC]
>  B -->|否| D[ETC2/EAC]
>  C --> E[按贴图类型选块大小]
>  D --> E
>```

>[!quote]- 通俗易懂的理解
>- 纹理压缩就是“用更省显存和带宽的格式存图”，前提是硬件能快速解码。
>- 移动端常用 ASTC/ETC2，本质是为了在有限功耗下保住画质和帧率。

>[!example]- 面试回答简版模板
>`纹理压缩选型本质是平台问题：桌面常见BC，移动端优先ASTC，兼容路径ETC2。目标是在可接受画质下减少显存占用和采样带宽。`

## 题目：远距离用高分辨率贴图会有什么问题？
>[!NOTE]- 标准准确的说法
>- **为什么这是问题**
>- 远处物体屏幕占比很小，一个像素覆盖的纹理区域很大。
>- 这时继续采高分辨率贴图，属于“读了很多用不上的细节”。
>- **直接后果**
>- 1) 带宽浪费、缓存命中下降。
>- 2) 高频细节超出采样能力，容易闪烁/摩尔纹。
>- 3) 功耗上升，帧时间不稳（移动端更明显）。
>- **正确做法**
>- 依赖 Mipmap + 正确 LOD；
>- 斜视角细节不足再配各向异性采样；
>- 结合纹理分辨率分档，避免资源过度配置。
>- **一句工程经验**
>- “看不见的细节不值得每帧花带宽去读”。

>[!tip]- 代码示例
>```cpp
>// 概念伪代码
>float lod = ComputeLOD(uv_ddx, uv_ddy);
>float3 color = SampleTextureTrilinear(tex, uv, lod);
>
>// 斜视角需要时提高各向异性
>color = ApplyAnisoIfNeeded(color, anisoLevel);
>```

>[!quote]- 通俗易懂的理解
>- 远处只占几个像素，却去读超高清贴图，像“用大卡车送一颗糖”。
>- 成本高、收益小，还可能更闪。

>[!example]- 面试回答简版模板
>`远距离继续采高分辨率贴图会浪费带宽并增加闪烁风险。正确做法是用Mipmap和LOD控制采样尺度，必要时叠加各向异性提升斜视角清晰度。`
