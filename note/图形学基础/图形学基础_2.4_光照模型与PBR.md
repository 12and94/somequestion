## 题目：PBR的原理？实时渲染中怎么体现？
>[!NOTE]- 标准准确的说法
>- **一句话定义**
>- PBR（Physically Based Rendering）是在“可实时预算”内，用符合物理规律的材质参数与反射模型近似求解渲染方程，让材质在不同光照环境下保持一致、可预测。
>- **理论基础（面试主干）**
>- 1) 渲染方程：出射光 = 自发光 + 入射光经 BRDF 加权积分。
>- 2) 微表面理论：表面由大量微小镜面组成，镜面反射由统计分布决定。
>- 3) 能量守恒：反射总能量不能超过入射能量。
>- 4) 菲涅尔效应：观察角越接近掠射角，反射越强。
>- 5) 参数语义化：材质参数尽量对应真实材料属性（而非纯经验调色）。
>- **实时最常见 BRDF 组合（Cook-Torrance）**
>- 镜面项：`f_spec = (D * F * G) / (4 * (N·L) * (N·V) + eps)`
>- 漫反射项：常用 Lambert：`f_diff = albedo / pi`
>- 能量分配：
>- `kS = F`
>- `kD = (1 - kS) * (1 - metallic)`
>- 单光源贡献常写为：
>- `Lo += (kD * albedo/pi + f_spec) * Li * max(N·L, 0)`
>- **实时渲染里的“体现方式”**
>- 1) **材质工作流**：主流是 Metallic-Roughness。
>- 输入常见为 `baseColor / metallic / roughness / normal / ao / emissive`。
>- `metallic` 决定“像金属还是介电体”，`roughness` 决定高光宽窄与模糊度。
>- 2) **直接光（Direct Lighting）**：逐灯评估 BRDF（方向光/点光/聚光）。
>- 3) **间接光（Image-Based Lighting, IBL）**：
>- 漫反射：用 irradiance map 近似半球积分。
>- 镜面：预滤波环境贴图（prefiltered env map）+ BRDF LUT（split-sum 近似）。
>- 4) **后处理闭环**：HDR 合成后做 Tone Mapping + Gamma 校正再输出到 LDR 显示设备。
>- **为什么 PBR 在工程上更稳**
>- 材质参数有物理语义，跨场景与跨光照条件迁移性更好。
>- 资产与灯光解耦更清晰，调参成本更可控。
>- 团队协作时“材质库可复用性”高于经验模型。
>- **高频工程细节（加分项）**
>- 粗糙度常做非线性映射（如 `alpha = roughness^2`）以匹配感知与分布函数。
>- 贴图色彩空间要分清：`baseColor` 常走 sRGB，`metallic/roughness/ao` 常走线性。
>- 法线贴图通常是切线空间法线，要经 TBN 变换到世界/观察空间参与光照。
>- 镜面抗锯齿、法线滤波、MIP 选择会显著影响 PBR 观感稳定性。
>- **边界与取舍**
>- 实时 PBR 不是“完整物理仿真”，而是可控近似。
>- 对多次反弹、焦散、复杂体积散射等，通常仍需额外近似或离线路径追踪补充。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[材质输入<br/>baseColor metallic roughness normal ao emissive] --> B[直接光: 逐灯Cook-Torrance]
>  A --> C[间接光: IBL]
>  C --> C1[Diffuse: Irradiance Map]
>  C --> C2[Specular: Prefilter Env + BRDF LUT]
>  B --> D[HDR合成]
>  C1 --> D
>  C2 --> D
>  D --> E[ToneMapping + Gamma]
>```
>
>```cpp
>// 伪代码（概念版）
>float3 N = normalize(worldNormal);
>float3 V = normalize(viewDir);
>float3 F0 = lerp(float3(0.04, 0.04, 0.04), baseColor, metallic);
>
>// 直接光
>float3 LoDirect = 0;
>for (Light light : lights) {
>    float3 L = normalize(light.dirToLight);
>    float3 H = normalize(V + L);
>    float NdotL = saturate(dot(N, L));
>    float NdotV = saturate(dot(N, V));
>
>    float  D  = D_GGX(N, H, roughness);
>    float3 F  = F_Schlick(saturate(dot(H, V)), F0);
>    float  G  = G_Smith(N, V, L, roughness);
>
>    float3 spec = (D * F * G) / max(4.0 * NdotL * NdotV, 1e-4);
>    float3 kS   = F;
>    float3 kD   = (1.0 - kS) * (1.0 - metallic);
>
>    LoDirect += (kD * baseColor / PI + spec) * light.radiance * NdotL;
>}
>
>// 间接光（split-sum 近似）
>float3 R = reflect(-V, N);
>float3 F = F_SchlickRoughness(saturate(dot(N, V)), F0, roughness);
>float3 kS = F;
>float3 kD = (1.0 - kS) * (1.0 - metallic);
>
>float3 irradiance = SampleIrradiance(N);
>float3 diffuseIBL = irradiance * baseColor;
>
>float3 prefiltered = SamplePrefilterEnv(R, roughness);
>float2 brdf = SampleBRDFLUT(saturate(dot(N, V)), roughness);
>float3 specIBL = prefiltered * (F * brdf.x + brdf.y);
>
>float3 LoIndirect = kD * diffuseIBL + specIBL;
>float3 colorHDR = LoDirect + LoIndirect + emissive;
>float3 colorLDR = GammaCorrect(ToneMap(colorHDR));
>```

>[!quote]- 通俗易懂的理解
>- 可以把 PBR 理解成“有物理约束的材质语言 + 一套统一算光流程”。
>- 你不再手调一堆“看起来像”的参数，而是用“金属度、粗糙度”这类有语义的参数控制材质。
>- 实时引擎里，直接光负责“灯打到物体”，IBL 负责“环境对物体的影响”，最后经 HDR 后处理落到屏幕。
>- 这就是为什么同一材质在白天、夜晚、室内、室外切换时，表现通常更稳定。

>[!example]- 面试回答简版模板
>`PBR的核心是用微表面BRDF（常见Cook-Torrance）+ 能量守恒 + 菲涅尔来近似渲染方程，并用 metallic/roughness 这类有物理语义的参数驱动材质。实时渲染里通常是逐灯计算直接光，再用 IBL（irradiance + prefilter env + BRDF LUT）近似间接光，最后做 Tone Mapping 与 Gamma。它比经验模型更稳定，能让材质在不同光照环境下保持一致性。`

## 题目：BRDF是什么？介绍一下
>[!NOTE]- 标准准确的说法
>- **定义**
>- BRDF（Bidirectional Reflectance Distribution Function）描述“入射方向光”在“观察方向”上反射多少。
>- **输入输出**
>- 输入：`N`（法线）、`L`（入射方向）、`V`（观察方向）。
>- 输出：该方向上的反射比例密度。
>- **性质（理想约束）**
>- 非负性。
>- 互易性（在适用前提下）。
>- 能量守恒。
>- **实时常见拆分**
>- 漫反射项 + 镜面项。
>- PBR 中镜面项常用 Cook-Torrance 形式。

>[!tip]- 代码示例
>```cpp
>// 概念表达：出射亮度 = 入射光 * BRDF * 几何项
>Lo += Li * f_brdf(N, V, L) * max(dot(N,L), 0.0);
>```

>[!quote]- 通俗易懂的理解
>- BRDF 就是“材质怎么反光”的规则函数。
>- 同样一束光打上去，塑料、金属、粗糙木头反出来不一样，本质差异就在 BRDF。

>[!example]- 面试回答简版模板
>`BRDF描述入射光在观察方向上的反射分布，是材质反光规律的数学模型。实时中通常分漫反射和镜面反射，且要满足能量守恒约束。`

## 题目：Cook-Torrance模型的各项含义？(D/F/G项)
>[!NOTE]- 标准准确的说法
>- **常见形式**
>- `spec = (D * F * G) / (4 * (N·L) * (N·V) + eps)`
>- **D 项（NDF）**
>- 法线分布函数，描述微表面法线朝向统计，决定高光形状与宽窄。
>- 常见选择：GGX。
>- **F 项（Fresnel）**
>- 描述反射率随角度变化，掠射角反射增强。
>- 实时常用 Schlick 近似。
>- **G 项（Geometry）**
>- 描述微表面遮蔽/阴影（masking-shadowing），控制可见微表面比例。
>- 实时常用 Smith 近似。
>- **参数影响**
>- roughness 同时影响 D 与 G。
>- F0 与入射角共同影响 F。

>[!tip]- 代码示例
>```mermaid
>flowchart LR
>  A[roughness] --> D[D项: 高光分布]
>  A --> G[G项: 遮蔽强度]
>  B[F0 + 角度] --> F[F项: 菲涅尔]
>  D --> S[Specular]
>  F --> S
>  G --> S
>```
>
>```cpp
>float D = D_GGX(N, H, roughness);
>float F = F_Schlick(F0, dot(H, V));
>float G = G_Smith(N, V, L, roughness);
>float3 spec = (D * F * G) / max(4 * NdotL * NdotV, 1e-4);
>```

>[!quote]- 通俗易懂的理解
>- D 决定“高光长什么样”，F 决定“斜着看有多反光”，G 决定“微结构互相挡住多少”。
>- 三项乘起来就是镜面反射的整体强度与形态。

>[!example]- 面试回答简版模板
>`Cook-Torrance中 D 控微表面分布，F 控角度相关反射，G 控遮蔽可见性。实时常见组合是 GGX + Schlick Fresnel + Smith Geometry。`

## 题目：Phong和Blinn-Phong的区别？
>[!NOTE]- 标准准确的说法
>- **Phong**
>- 用反射向量 `R` 与视线 `V` 的点积控制高光：`pow(max(dot(R,V),0), shininess)`。
>- **Blinn-Phong**
>- 用半程向量 `H = normalize(L+V)` 与法线 `N` 的点积：`pow(max(dot(N,H),0), shininess)`。
>- **差异**
>- Blinn-Phong 在数值稳定性和实现便利性上通常更好。
>- 同样指数参数下，两者高光外观并不完全一致。
>- 二者本质都是经验模型，不严格物理。

>[!tip]- 代码示例
>```cpp
>// Phong
>float specP = pow(max(dot(R, V), 0.0), shininess);
>
>// Blinn-Phong
>float3 H = normalize(L + V);
>float specB = pow(max(dot(N, H), 0.0), shininess);
>```

>[!quote]- 通俗易懂的理解
>- Phong 像“看反射光线有没有对准眼睛”，Blinn-Phong 像“看法线和中间方向贴不贴”。

>[!example]- 面试回答简版模板
>`Blinn-Phong把Phong的反射向量改成半程向量，通常更稳定、更易用；但两者都偏经验模型，物理一致性不如PBR。`

## 题目：菲涅尔项的F0有什么含义？
>[!NOTE]- 标准准确的说法
>- **定义**
>- `F0` 是法线入射（接近垂直入射）时的基础反射率。
>- **材质含义**
>- 介电体（塑料/木头/皮肤）F0 通常较低且接近中性色。
>- 金属 F0 更高且常带材质色（有色镜面）。
>- **在 metallic-roughness 工作流中的角色**
>- metallic 影响“高光颜色来源”和“漫反射占比”。
>- F0 是镜面反射强度/颜色的基准。
>- **角度关系**
>- 随观察角趋向掠射，反射增强（Schlick 近似常见）。

>[!tip]- 代码示例
>```cpp
>// Schlick近似
>float3 FresnelSchlick(float cosTheta, float3 F0) {
>    return F0 + (1.0 - F0) * pow(1.0 - cosTheta, 5.0);
>}
>```

>[!quote]- 通俗易懂的理解
>- F0 可以理解为“正面看时，这种材质先天有多镜面”。
>- 之后角度越斜，菲涅尔会把反射再往上抬。

>[!example]- 面试回答简版模板
>`F0是正入射时的基础反射率，决定镜面反射底强度与底色。PBR里它与metallic一起决定金属/非金属的高光分配。`

## 题目：什么是能量守恒？
>[!NOTE]- 标准准确的说法
>- **定义**
>- 着色中的能量守恒指：反射出去的总能量不能超过入射能量。
>- **实现思路**
>- 常见通过限制漫反射与镜面反射权重关系实现，如 `kD + kS <= 1`。
>- 在 metallic-roughness 里，metallic 和 Fresnel 会共同影响这类分配。
>- **不守恒会怎样**
>- 多光源下“越照越亮”、材质发白、视觉不可信。

>[!tip]- 代码示例
>```cpp
>float3 kS = F;              // 镜面占比
>float3 kD = (1.0 - kS);     // 漫反射占比
>kD *= (1.0 - metallic);     // 金属几乎无漫反射
>```

>[!quote]- 通俗易懂的理解
>- 你不能让材质“凭空生光”。
>- 它最多把收到的光重新分配出去，不该无上限变亮。

>[!example]- 面试回答简版模板
>`能量守恒是限制反射总能量不超过入射能量。工程上通常通过漫反射与镜面反射权重约束实现，避免材质在多光源下不合理增亮。`

## 题目：渲染方程的物理意义？
>[!NOTE]- 标准准确的说法
>- **物理意义**
>- 渲染方程描述某点向观察方向的出射辐射亮度：
>- 自发光 + 入射半球上的反射积分。
>- **统一性**
>- 把局部反射模型（BRDF）与全局光传输（多次反弹）统一在一个框架里。
>- **实时中的近似**
>- 直接光：逐灯近似。
>- 间接光：IBL、探针、SSGI 等近似。
>- 全精度解通常依赖路径追踪/离线方法。

>[!tip]- 代码示例
>```cpp
>// 概念表达（非完整积分实现）
>Lo = Le + Integral_hemisphere( Li * f_r * cosTheta * dOmega );
>```
>
>```mermaid
>flowchart LR
>  A[自发光 Le] --> D[出射亮度 Lo]
>  B[入射光 Li] --> C[BRDF加权积分]
>  C --> D
>```

>[!quote]- 通俗易懂的理解
>- 一个像素有多亮，不仅看“灯直接照到多少”，还看“环境里反弹来的光”有多少。

>[!example]- 面试回答简版模板
>`渲染方程描述出射光=自发光+入射光经BRDF加权积分，是局部与全局光照统一表达。实时渲染通常用直接光+IBL等近似来平衡效果与性能。`

## 题目：局部光照和全局光照的方案？
>[!NOTE]- 标准准确的说法
>- **局部光照（Local Illumination）**
>- 通常只考虑光源到表面的一次作用，忽略多次反弹。
>- 代表：Phong/Blinn/PBR直接光部分。
>- **全局光照（Global Illumination, GI）**
>- 考虑间接反射、颜色渗透、软阴影等多次传输。
>- **实时 GI 常见方案**
>- IBL（环境贴图）
>- Light Probe / Reflection Probe
>- SSGI
>- DDGI / VXGI / Lumen 类
>- **离线高精度**
>- 路径追踪（Path Tracing）等。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[直接光 Local] --> C[最终光照]
>  B[间接光 GI近似] --> C
>```

>[!quote]- 通俗易懂的理解
>- 局部光照像“只算直射灯”，全局光照像“把光在房间里弹来弹去也算进去”。

>[!example]- 面试回答简版模板
>`局部光照成本低但缺少间接信息；全局光照更真实但代价高。实时项目通常用直接光+IBL/探针等近似方案，再按预算叠加更高阶GI。`

## 题目：法线分布函数有哪些？
>[!NOTE]- 标准准确的说法
>- **NDF 作用**
>- NDF（Normal Distribution Function）描述微表面法线朝向统计，直接影响高光形状与尾部。
>- **常见类型**
>- GGX / Trowbridge-Reitz：
>- 尾部更长，粗糙材质观感自然，现代实时 PBR 最常见。
>- Beckmann：
>- 历史较早，分布特性更接近高斯类。
>- Blinn（幂指数分布）：
>- 常见于传统高光模型体系。
>- **选型影响**
>- 改变 NDF 会改变高光“宽窄、拖尾、能量分布”，直接影响材质观感。

>[!tip]- 代码示例
>```cpp
>// 概念：不同NDF返回的D值不同
>float D_ggx      = NDF_GGX(N, H, roughness);
>float D_beckmann = NDF_Beckmann(N, H, roughness);
>```

>[!quote]- 通俗易懂的理解
>- NDF 就是“表面上无数小镜子的朝向统计图”。
>- 统计分布不同，高光看起来就会不一样：有的短促，有的拖尾更长。

>[!example]- 面试回答简版模板
>`NDF描述微表面法线分布，决定高光形态。实时PBR中GGX最常见，因为它对粗糙材质的高光尾部表现更自然。`
