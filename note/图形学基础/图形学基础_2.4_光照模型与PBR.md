## 题目：PBR的原理？实时渲染中怎么体现？
>[!NOTE]- 标准准确的说法
>- **一句话定义**
>- PBR（Physically Based Rendering）是在“可实时预算”内，用符合物理规律的材质参数与反射模型近似求解渲染方程，让材质在不同光照环境下保持一致、可预测。
>- **理论基础（面试主干）**
>- 1) 渲染方程：出射光 = 自发光 + 入射光经 BRDF 加权积分。
>- 2) 微表面理论：表面由大量微小镜面组成，镜面反射由统计分布决定。
>- 3) 能量守恒：反射总能量不能超过入射能量。
>- 4) 菲涅尔效应：观察角越接近掠射角，反射越强。
>- 5) 参数语义化：材质参数尽量对应真实材料属性（而非纯经验调色）。
>- **实时最常见 BRDF 组合（Cook-Torrance）**
>- 镜面项：
> $$
> f_{spec} = \frac{D \cdot F \cdot G}{4 (N \cdot L)(N \cdot V) + \varepsilon}
> $$
>- 漫反射项（Lambert）：
> $$
> f_{diff} = \frac{albedo}{\pi}
> $$
>- 能量分配：
> $$
> k_S = F,\quad
> k_D = (1-k_S)(1-metallic)
> $$
>- 单光源贡献常写为：
> $$
> L_o \mathrel{+}= \left(k_D \frac{albedo}{\pi} + f_{spec}\right)L_i\max(N\cdot L,0)
> $$
>- **实时渲染里的“体现方式”**
>- 1) **材质工作流**：主流是 Metallic-Roughness。
>- 输入常见为 `baseColor / metallic / roughness / normal / ao / emissive`。
>- `metallic` 决定“像金属还是介电体”，`roughness` 决定高光宽窄与模糊度。
>- 2) **直接光（Direct Lighting）**：逐灯评估 BRDF（方向光/点光/聚光）。
>- 3) **间接光（Image-Based Lighting, IBL）**：
>- 漫反射：用 irradiance map 近似半球积分。
>- 镜面：预滤波环境贴图（prefiltered env map）+ BRDF LUT（split-sum 近似）。
>- 4) **后处理闭环**：HDR 合成后做 Tone Mapping + Gamma 校正再输出到 LDR 显示设备。
>- **为什么 PBR 在工程上更稳**
>- 材质参数有物理语义，跨场景与跨光照条件迁移性更好。
>- 资产与灯光解耦更清晰，调参成本更可控。
>- 团队协作时“材质库可复用性”高于经验模型。
>- **高频工程细节（加分项）**
>- 粗糙度常做非线性映射（如 $\alpha = roughness^2$）以匹配感知与分布函数。
>- 贴图色彩空间要分清：`baseColor` 常走 sRGB，`metallic/roughness/ao` 常走线性。
>- 法线贴图通常是切线空间法线，要经 TBN 变换到世界/观察空间参与光照。
>- 镜面抗锯齿、法线滤波、MIP 选择会显著影响 PBR 观感稳定性。
>- **边界与取舍**
>- 实时 PBR 不是“完整物理仿真”，而是可控近似。
>- 对多次反弹、焦散、复杂体积散射等，通常仍需额外近似或离线路径追踪补充。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[材质输入<br/>baseColor metallic roughness normal ao emissive] --> B[直接光: 逐灯Cook-Torrance]
>  A --> C[间接光: IBL]
>  C --> C1[Diffuse: Irradiance Map]
>  C --> C2[Specular: Prefilter Env + BRDF LUT]
>  B --> D[HDR合成]
>  C1 --> D
>  C2 --> D
>  D --> E[ToneMapping + Gamma]
>```
>
>```cpp
>// 伪代码（概念版）
>float3 N = normalize(worldNormal);
>float3 V = normalize(viewDir);
>float3 F0 = lerp(float3(0.04, 0.04, 0.04), baseColor, metallic);
>
>// 直接光
>float3 LoDirect = 0;
>for (Light light : lights) {
>    float3 L = normalize(light.dirToLight);
>    float3 H = normalize(V + L);
>    float NdotL = saturate(dot(N, L));
>    float NdotV = saturate(dot(N, V));
>
>    float  D  = D_GGX(N, H, roughness);
>    float3 F  = F_Schlick(saturate(dot(H, V)), F0);
>    float  G  = G_Smith(N, V, L, roughness);
>
>    float3 spec = (D * F * G) / max(4.0 * NdotL * NdotV, 1e-4);
>    float3 kS   = F;
>    float3 kD   = (1.0 - kS) * (1.0 - metallic);
>
>    LoDirect += (kD * baseColor / PI + spec) * light.radiance * NdotL;
>}
>
>// 间接光（split-sum 近似）
>float3 R = reflect(-V, N);
>float3 F = F_SchlickRoughness(saturate(dot(N, V)), F0, roughness);
>float3 kS = F;
>float3 kD = (1.0 - kS) * (1.0 - metallic);
>
>float3 irradiance = SampleIrradiance(N);
>float3 diffuseIBL = irradiance * baseColor;
>
>float3 prefiltered = SamplePrefilterEnv(R, roughness);
>float2 brdf = SampleBRDFLUT(saturate(dot(N, V)), roughness);
>float3 specIBL = prefiltered * (F * brdf.x + brdf.y);
>
>float3 LoIndirect = kD * diffuseIBL + specIBL;
>float3 colorHDR = LoDirect + LoIndirect + emissive;
>float3 colorLDR = GammaCorrect(ToneMap(colorHDR));
>```

>[!quote]- 通俗易懂的理解
>- 可以把 PBR 理解成“有物理约束的材质语言 + 一套统一算光流程”。
>- 你不再手调一堆“看起来像”的参数，而是用“金属度、粗糙度”这类有语义的参数控制材质。
>- 实时引擎里，直接光负责“灯打到物体”，IBL 负责“环境对物体的影响”，最后经 HDR 后处理落到屏幕。
>- 这就是为什么同一材质在白天、夜晚、室内、室外切换时，表现通常更稳定。

>[!example]- 面试回答简版模板
>`PBR的核心是用微表面BRDF（常见Cook-Torrance）+ 能量守恒 + 菲涅尔来近似渲染方程，并用 metallic/roughness 这类有物理语义的参数驱动材质。实时渲染里通常是逐灯计算直接光，再用 IBL（irradiance + prefilter env + BRDF LUT）近似间接光，最后做 Tone Mapping 与 Gamma。它比经验模型更稳定，能让材质在不同光照环境下保持一致性。`

## 题目：BRDF是什么？介绍一下
>[!NOTE]- 标准准确的说法
>- **定义（严格版）**
>- BRDF（Bidirectional Reflectance Distribution Function）描述在某个表面点上，入射方向 `wi` 的辐照如何分配到出射方向 `wo` 的辐亮度。
>- 数学上可写为：
> $$
> f_r(\omega_i,\omega_o)=\frac{dL_o(\omega_o)}{L_i(\omega_i)\cos\theta_i\,d\omega_i}
> $$
>- 可理解为“单位入射能量，在某观察方向上反射了多少”。
>- **输入/输出与单位**
>- 输入：法线 `N`、入射方向 `L(wi)`、观察方向 `V(wo)`，常通过 `N·L / N·V / N·H / V·H` 等内积参与计算。
>- 输出：方向相关的反射分布值（单位常见 `1/sr` 语义）。
>- **核心物理约束（面试高频）**
>- 1) 非负性：$f_r \ge 0$。
>- 2) 互易性（Helmholtz reciprocity）：$f_r(\omega_i,\omega_o)=f_r(\omega_o,\omega_i)$（在满足条件的介质中）。
>- 3) 能量守恒：半球积分不应超过 1。
>- **和相关概念的区别**
>- BRDF：只描述“表面反射”。
>- BSDF：反射 + 透射（更一般）。
>- BSSRDF：可描述次表面散射（皮肤、蜡等）。
>- **实时渲染中的常见形态**
>- 漫反射部分：Lambert / Oren-Nayar 等。
>- 镜面部分：Cook-Torrance（D/F/G）最常见。
>- 实际着色是 BRDF 与光源辐照、几何项共同作用的结果。
>- **为什么 BRDF 很关键**
>- 它是“材质长相”的核心函数：同样光照条件下，材质差异本质由 BRDF 决定。
>- 现代 PBR 的稳定性很大程度来自 BRDF 的物理约束与可解释参数化。

>[!tip]- 代码示例
>```cpp
>// 概念表达：逐光源近似渲染方程中的 BRDF 项
>float3 EvalLighting(float3 N, float3 V, float3 L, float3 Li) {
>    float3 fr = EvalBRDF(N, V, L); // e.g. Lambert + Cook-Torrance
>    float  NdotL = max(dot(N, L), 0.0);
>    return Li * fr * NdotL;
>}
>```
>
>```mermaid
>flowchart LR
>  A[入射光 Li(wi)] --> B[BRDF fr(wi,wo)]
>  C[观察方向 wo] --> B
>  D[法线与余弦项 cos(theta)] --> B
>  B --> E[出射光 Lo(wo)]
>```

>[!quote]- 通俗易懂的理解
>- BRDF 可以理解成“材质的反光规则表”。
>- 光从哪里来、你从哪里看，规则表会告诉你“这个方向该反多少光”。
>- 金属、塑料、木头看起来不同，本质就是它们的 BRDF 不同。

>[!example]- 面试回答简版模板
>`BRDF描述表面点处入射光到出射光的方向分配关系，是材质反光规律的核心函数。它输入入射/观察方向与法线，输出方向相关反射强度，并应满足非负、互易和能量守恒。实时中常把 BRDF 拆为漫反射+镜面反射，镜面常用 Cook-Torrance。`

## 题目：Cook-Torrance模型的各项含义？(D/F/G项)
>[!NOTE]- 标准准确的说法
>- **先澄清边界（很关键）**
>- 严格来说，Cook-Torrance 在实时 PBR 里通常指“微表面镜面 BRDF 项”，不是完整的最终出射光模型。
>- 实际逐光源着色常是“漫反射 + Cook-Torrance 镜面项”的组合。
>- **Cook-Torrance 标准形式（镜面项）**
> $$
> f_{spec} = \frac{D\cdot F\cdot G}{4(N\cdot L)(N\cdot V)+\varepsilon}
> $$
>- 其中 `N` 法线，`L` 入射方向，`V` 观察方向，半程向量为 $H = normalize(L+V)$。
>- **完整 PBR BRDF（常见写法）**
> $$
> f_r = k_D\frac{albedo}{\pi} + f_{spec}
> $$
> $$
> k_S = F,\quad
> k_D = (1-k_S)(1-metallic)
> $$
>- 对应的单光源贡献常写为：
> $$
> L_o = f_r \, L_i \, \max(N\cdot L, 0)
> $$
>- **D 项：NDF（Normal Distribution Function）**
>- 描述微表面法线朝向统计，决定高光“宽窄、拖尾、峰值形状”。
>- 实时最常用 GGX/Trowbridge-Reitz，原因是粗糙表面高光尾部更自然。
>- 典型实现会把 `roughness` 映射到 $\alpha = roughness^2$ 再入公式，控制更稳定。
>- **F 项：Fresnel（菲涅尔）**
>- 描述反射率随角度变化：掠射角反射增强。
>- 实时常用 Schlick 近似：
> $$
> F = F_0 + (1-F_0)(1-V\cdot H)^5
> $$
>- `F0` 决定正入射基础反射率（介电体常约 0.04，金属通常更高且有色）。
>- **G 项：Geometry（几何遮蔽）**
>- 描述微表面间的遮蔽/阴影（masking-shadowing），控制真正“可见且可反射”的微面比例。
>- 实时常用 Smith 几何项（配合 GGX 的 Schlick-GGX 近似）。
>- **三项耦合关系（面试加分）**
>- `roughness` 同时影响 D 与 G，既影响高光扩散，也影响几何遮挡。
>- `F0` 与角度主导 F，决定高光底色与角度增强幅度。
>- 分母 $4(N\cdot L)(N\cdot V)$ 是几何归一化因子，缺失会导致能量异常。
>- **常见工程坑**
>- 1) 忘记 clamp：`N·L`、`N·V` 负值会造成错误。
>- 2) 分母未加 `eps` 可能数值爆炸。
>- 3) `roughness` 不做平方映射会导致手感与美术预期不一致。
>- 4) 法线贴图高频区域会出现镜面闪烁，需要 specular AA / 正确 MIP 策略。

>[!tip]- 代码示例
>```cpp
>float3 H = normalize(V + L);
>float NdotL = saturate(dot(N, L));
>float NdotV = saturate(dot(N, V));
>float NdotH = saturate(dot(N, H));
>float VdotH = saturate(dot(V, H));
>
>float alpha = roughness * roughness;
>float D = D_GGX(NdotH, alpha);
>float3 F = F_Schlick(VdotH, F0);
>float G = G_Smith_SchlickGGX(NdotV, NdotL, alpha);
>
>float3 fSpec = (D * F * G) / max(4.0 * NdotL * NdotV, 1e-4); // Cook-Torrance镜面项
>float3 kS = F;
>float3 kD = (1.0 - kS) * (1.0 - metallic);
>
>// 完整逐光源BRDF = 漫反射 + 镜面
>float3 fBRDF = kD * baseColor / PI + fSpec;
>float3 Lo = fBRDF * Li * NdotL;
>```
>
>```mermaid
>flowchart LR
>  A[roughness] --> D[D: 法线分布]
>  A --> G[G: 几何遮蔽]
>  B[F0 + 角度] --> F[F: 菲涅尔]
>  D --> S[Specular BRDF]
>  F --> S
>  G --> S
>  S --> M[与漫反射合成完整BRDF]
>```

>[!quote]- 通俗易懂的理解
>- D 决定“高光长什么样”，F 决定“角度变斜时反光涨多少”，G 决定“微结构自己挡掉多少光”。
>- 这三项先组成“镜面反射规则”；最终画面还要再和漫反射项合成，才是完整 BRDF。

>[!example]- 面试回答简版模板
>`Cook-Torrance通常指微表面镜面项 f_spec=(D*F*G)/(4NdotLNdotV)，其中 D/F/G 分别对应法线分布、菲涅尔和几何遮蔽。完整实时PBR通常写成 f_r = kD*albedo/pi + f_spec，再乘 Li 和 NdotL 得到单光源贡献。`

## 题目：Phong和Blinn-Phong的区别？
>[!NOTE]- 标准准确的说法
>- **两者共同点**
>- 都是经验型局部光照模型，用幂函数控制高光锐利度，计算成本低、实现简单。
>- **Phong 镜面项**
>- 先算反射向量 $R=reflect(-L,N)$，再算 $\max(R\cdot V,0)^{shininess}$。
>- 高光“是否对准视线”取决于反射向量与视线夹角。
>- **Blinn-Phong 镜面项**
>- 用半程向量 $H=normalize(L+V)$，高光项为 $\max(N\cdot H,0)^{shininess}$。
>- 避免直接构造反射向量，数值上通常更稳，移动端实现更常见。
>- **关键差异**
>- 1) 几何解释不同：Phong 看 $R\cdot V$，Blinn 看 $N\cdot H$。
>- 2) 指数参数不可直接等价：相同 shininess 下视觉高光宽度不同。
>- 3) 当 `V` 与 `R`接近反向时，Phong 更容易出现高光突变。
>- 4) 二者都缺乏完整物理约束（能量守恒、菲涅尔、微表面遮蔽等表达不足）。
>- **和 PBR 的关系**
>- Phong/Blinn-Phong 适合低成本、风格化、老项目兼容。
>- 在追求跨环境一致性与材质可解释性时，通常升级到 PBR（Cook-Torrance）。
>- **工程常见实践**
>- 老项目保留 Blinn-Phong 作为 fallback，主路径走 PBR。
>- 对艺术风格化项目，可将 Blinn-Phong 与 toon/ramp 结合做风格高光。

>[!tip]- 代码示例
>```cpp
>// Phong
>float3 R = reflect(-L, N);
>float specPhong = pow(max(dot(R, V), 0.0), shininessPhong);
>
>// Blinn-Phong
>float3 H = normalize(L + V);
>float specBlinn = pow(max(dot(N, H), 0.0), shininessBlinn);
>```
>
>```text
>经验上常需要重新标定指数：
>同观感下，Blinn-Phong 的指数通常要高于 Phong（常见近似约 2~4 倍区间）。
>```

>[!quote]- 通俗易懂的理解
>- Phong 像“看镜子反射光有没有正好打到眼睛”。
>- Blinn-Phong 像“看法线与光-眼中间方向贴不贴”。
>- 所以 Blinn 通常更稳一些，但两者都属于“好用但不太物理”的老牌方案。

>[!example]- 面试回答简版模板
>`Phong 用 R·V，Blinn-Phong 用 N·H。Blinn-Phong 通常更稳定、实现更方便，但两者都属于经验模型，物理一致性弱于 PBR。工程上常把它们作为低成本或兼容路径，主渲染链路通常采用 PBR。`

## 题目：菲涅尔项的F0有什么含义？
>[!NOTE]- 标准准确的说法
>- **F0 的定义**
>- `F0` 是法线正入射（入射角接近 0°）时的基础反射率，即“材质正面看上去的镜面底反射”。
>- **物理来源**
>- 对理想介电体，`F0` 可由折射率近似推得：
>- $F_0 = \left(\frac{\eta-1}{\eta+1}\right)^2$（单通道简化表达）。
>- 实际渲染会用 RGB 三通道以表达有色反射。
>- **典型数值认知**
>- 介电体（塑料、木头、皮肤等）F0 常较低，工程上常用 `~0.04` 作为默认近似。
>- 金属 F0 通常更高且有颜色（铜偏红、金偏黄等）。
>- **在 Metallic-Roughness 工作流中的角色**
>- 常见公式：$F_0 = lerp(0.04, baseColor, metallic)$。
>- $metallic=0$ 时，镜面底色接近中性低反射；
>- $metallic=1$ 时，镜面颜色主要来自 baseColor，漫反射基本消失。
>- **角度关系（与 Schlick）**
>- 实时常用：$F = F_0 + (1-F_0)(1-cos\theta)^5$。
>- 含义：无论 F0 多低，掠射角反射都会显著增强。
>- **常见误区**
>- 1) 把 F0 当“整体亮度旋钮”乱调，会破坏材质物理语义。
>- 2) 介电体给过高 F0 会出现“塑料像金属”的假感。
>- 3) 忽略贴图色彩空间会让 F0 与 baseColor 混合结果失真。

>[!tip]- 代码示例
>```cpp
>// metallic-roughness 常见写法
>float3 dielectricF0 = float3(0.04, 0.04, 0.04);
>float3 F0 = lerp(dielectricF0, baseColor, metallic);
>
>// Schlick Fresnel
>float3 FresnelSchlick(float cosTheta, float3 F0) {
>    return F0 + (1.0 - F0) * pow(1.0 - cosTheta, 5.0);
>}
>```
>
>```text
>理解要点：
>F0 决定“正面镜面反射起点”，角度项决定“斜着看时反射抬升幅度”。
>```

>[!quote]- 通俗易懂的理解
>- F0 可以理解为“材质的先天镜面底色/底强度”。
>- 正面看时它主导高光基线；越斜看，菲涅尔会把反射继续抬高。
>- 所以 F0 负责“底子”，角度负责“涨幅”。

>[!example]- 面试回答简版模板
>`F0 是正入射时的基础反射率，决定镜面反射的底强度与底色。PBR 的 metallic-roughness 工作流常用 $F_0=lerp(0.04, baseColor, metallic)$，再结合 Schlick 菲涅尔实现角度相关反射增强。`

## 题目：什么是能量守恒？
>[!NOTE]- 标准准确的说法
>- **定义**
>- 在着色语境中，能量守恒指材料反射/透射出去的总能量不能超过接收到的入射能量。
>- 对纯反射 BRDF，可理解为半球积分不应大于 1。
>- **实时渲染中的落地约束**
>- 常见做法：控制漫反射与镜面反射分配，使 $k_D + k_S \le 1$（按通道考虑）。
>- 在 metallic-roughness 里，常写：
> $$
> k_S = F,\quad
> k_D = (1-k_S)(1-metallic)
> $$
>- 含义是：金属主要走镜面，介电体保留漫反射份额。
>- **为什么必须做这件事**
>- 多光源叠加、强环境光、HDR 场景下，如果不守恒会“越叠越亮”，材质发白、发灰、丢层次。
>- **和粗糙度/多次散射的关系**
>- 简化实时模型常只建模一次反射，粗糙表面的多次微表面散射会导致能量偏差。
>- 工程上可通过经验补偿（如环境镜面补偿）减少“高 roughness 下过暗/过亮”问题。
>- **常见误区**
>- 1) 把 AO 直接乘到所有项导致过度压暗（通常更偏向影响间接光）。
>- 2) 不区分线性空间与 gamma 空间，导致能量关系被破坏。
>- 3) 手动叠加多个“艺术增亮项”后忘记统一归一化。

>[!tip]- 代码示例
>```cpp
>// 典型能量分配
>float3 kS = FresnelSchlick(cosTheta, F0); // 镜面份额
>float3 kD = (1.0 - kS) * (1.0 - metallic); // 漫反射份额
>
>float3 diffuse = kD * baseColor / PI;
>float3 specular = CookTorranceSpec(...);
>float3 Lo = (diffuse + specular) * Li * NdotL;
>```
>
>```text
>判断是否守恒的直观信号：
>多灯环境中材质是否出现异常“泛白发光”、层次崩掉。
>```

>[!quote]- 通俗易懂的理解
>- 材质不是灯泡，不能“凭空变亮”。
>- 它只能把收到的光按一定比例重新分配出去。
>- 能量守恒就是给这套分配规则加上“总预算上限”。

>[!example]- 面试回答简版模板
>`能量守恒要求反射总能量不超过入射能量。实时PBR里常通过 $k_S/k_D$ 分配控制（如 $k_D=(1-k_S)(1-metallic)$）来约束漫反射与镜面反射，避免多光源和HDR场景下材质异常增亮。`

>[!info]- 对应示例
>- `note/图形学基础/图形学基础_2.4_光照模型与PBR.md`

## 题目：渲染方程的物理意义？
>[!NOTE]- 标准准确的说法
>- **标准表达（表面点 x，观察方向 wo）**
> $$
> L_o(x,\omega_o)=L_e(x,\omega_o)+\int_{\Omega} f_r(x,\omega_i,\omega_o)\,L_i(x,\omega_i)\,\max(N\cdot\omega_i,0)\,d\omega_i
> $$
>- **每一项的物理意义**
>- `Le`：自发光（材质自己发出的光）。
>- `Li`：来自入射方向 `wi` 的入射辐亮度。
>- `fr`：BRDF，决定材质如何把入射光分配到观察方向。
>- `max(N·wi,0)`：余弦项，体现入射角有效投影面积。
>- 积分：表示要把半球上所有方向的入射贡献都累加。
>- **为什么它重要**
>- 它统一了“局部反射模型”与“全局光传输”。
>- 阴影、间接反弹、颜色渗透、本质上都在这个框架里。
>- **为什么难直接精确求解**
>- `Li` 往往又依赖其他点的 `Lo`，形成递归耦合问题。
>- 场景复杂时解析解不可得，需数值近似（蒙特卡洛、路径追踪或实时近似）。
>- **实时渲染中的近似拆分**
>- 直接光：逐灯近似积分中的“显式光源项”。
>- 间接光：IBL、探针、SSGI、DDGI、RTGI 等近似。
>- 后处理：Tone Mapping 把 HDR 动态范围映射到显示设备。
>- **面试加分视角**
>- PBR 并不是“另一个独立体系”，而是渲染方程在实时条件下的一组工程化近似。

>[!tip]- 代码示例
>```cpp
>// 概念化伪代码：实时中的“分治近似”
>float3 Lo = Emissive;
>Lo += EvalDirectLighting_BRDF(...);   // 直接光
>Lo += EvalIndirectLighting_IBL(...);  // 间接光近似
>Lo = ToneMapAndGamma(Lo);             // 显示映射
>```
>
>```mermaid
>flowchart LR
>  A[自发光 Le] --> D[Lo]
>  B[入射光 Li(半球)] --> C[BRDF与余弦加权积分]
>  C --> D
>```

>[!quote]- 通俗易懂的理解
>- 渲染方程在回答一个问题：
>- “你眼里看到的这点亮度，来自它自己发光，还是来自周围所有方向打来的光再反射过来？”
>- 它是光照计算的总账本，实时渲染是在预算内做近似记账。

>[!example]- 面试回答简版模板
>`渲染方程给出出射光的统一表达：$L_o=L_e+\int_{\Omega}L_i f_r \cos\theta \,d\omega$。它统一了局部与全局光照，但精确求解代价高。实时引擎通常把它拆成逐灯直接光+IBL/探针等间接光近似，再通过 ToneMapping 输出。`

>[!info]- 对应示例
>- `note/图形学基础/图形学基础_2.4_光照模型与PBR.md`

## 题目：局部光照和全局光照的方案？
>[!NOTE]- 标准准确的说法
>- **局部光照（Local Illumination）**
>- 主要考虑“光源 -> 当前表面”的一次作用，不显式建模多次反弹。
>- 常见形态：Phong/Blinn-Phong、PBR 的直接光部分、阴影贴图直射项。
>- 优点：成本低、稳定、易控；缺点：缺少环境反弹与真实空间耦合。
>- **全局光照（Global Illumination, GI）**
>- 考虑多次反射、颜色渗透、间接阴影、软阴影等全局传输效应。
>- 画面真实感显著提升，但计算与内存成本更高。
>- **实时项目常见方案分层**
>- 1) 低成本近似：
>- IBL（天空盒/环境图）
>- AO（SSAO/GTAO）增强接触阴影感
>- 反射探针（Reflection Probe）
>- 2) 中成本方案：
>- Lightmap（静态烘焙 GI）
>- Light Probe/SH（动态物体采样静态间接光）
>- SSGI/SSR（屏幕空间近似）
>- 3) 高成本方案：
>- DDGI/VXGI/SDF-GI/Lumen 类
>- RTGI（硬件光追）
>- **典型工程组合（而非单一方案）**
>- 静态场景：Lightmap + Probe + 直接光。
>- 开放世界：IBL + Probe + 局部实时 GI（如 SSGI/DDGI/Lumen）。
>- 高端平台：PBR + 阴影 + 反射 + RTGI 混合路径。
>- **离线高精度**
>- 路径追踪/双向路径追踪等，可逼近全局解，但实时性弱。
>- **选型思路（面试加分）**
>- 先看平台预算（GPU、内存、功耗）与场景动态性（静态/动态比例）。
>- 再看目标画质（写实 vs 风格化）和团队管线成熟度（烘焙/探针/光追工具链）。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[直接光 Local] --> D[最终结果]
>  B[IBL/Probe/Lightmap] --> D
>  C[SSGI/DDGI/RTGI 等动态GI] --> D
>```
>
>工程常见思路：
> $$
> FinalLighting = Direct + BakedGI + DynamicGI + Reflection
> $$

>[!quote]- 通俗易懂的理解
>- 局部光照像“只算灯直接打到你身上的光”。
>- 全局光照像“房间里所有墙面反弹回来的光也算进去”。
>- 真项目通常不是二选一，而是按预算把它们拼成一套混合方案。

>[!example]- 面试回答简版模板
>`局部光照主要算直射，成本低但缺少反弹信息；全局光照考虑多次传输，真实但代价高。实时项目通常采用分层混合：Direct + IBL/Probe/Lightmap，再按预算叠加 SSGI、DDGI、Lumen 或 RTGI。`

>[!info]- 对应示例

## 题目：法线分布函数有哪些？
>[!NOTE]- 标准准确的说法
>- **NDF 的作用**
>- NDF（Normal Distribution Function）描述微表面法线朝向的统计分布，直接决定镜面高光的形状、宽度与尾部分布。
>- 在 Cook-Torrance 中，NDF 对应 `D` 项。
>- **常见 NDF 类型**
>- 1) GGX / Trowbridge-Reitz（现代实时最常见）
>- 特点：高光尾部更长，粗糙材质观感自然，适配金属/塑料等大多数写实材质。
>- 2) Beckmann
>- 特点：分布更“收敛”，历史上常用于离线/传统模型，粗糙高光尾部相对短。
>- 3) Blinn-Phong NDF（幂指数族）
>- 特点：实现简单，常见于旧式高光模型，与现代 PBR 参数语义对齐较弱。
>- 4) Charlie（常用于布料）
>- 特点：适合织物类“宽而柔”的高光分布（cloth BRDF 场景常见）。
>- **为什么实时里 GGX 占主流**
>- 视觉上更贴近真实粗糙表面。
>- 与 Smith 几何项、Schlick 菲涅尔等组合成熟，工程生态完整。
>- 美术调参手感稳定，跨材质迁移性好。
>- **参数与映射**
>- 常见做法 $\alpha = roughness^2$，因为感知线性 roughness 直接入公式容易导致分布不均衡。
>- **工程问题与应对**
>- 高光走样/闪烁：需要 specular anti-aliasing、法线过滤与正确 MIP。
>- 高法线频率资产：切线空间法线与 NDF 叠加后更易出噪点。
>- 统一材质观感：同项目内尽量统一 NDF 族，避免跨材质高光语义不一致。

>[!tip]- 代码示例
>```cpp
>// 概念：同一 N,H,roughness 下，不同 NDF 得到不同 D
>float alpha = roughness * roughness;
>float D_ggx      = NDF_GGX(NdotH, alpha);
>float D_beckmann = NDF_Beckmann(NdotH, alpha);
>float D_blinn    = NDF_BlinnPhong(NdotH, shininess);
>```
>
>```mermaid
>flowchart LR
>  A[NDF选择] --> B[高光峰值]
>  A --> C[高光宽度]
>  A --> D[高光尾部]
>  B --> E[最终材质观感]
>  C --> E
>  D --> E
>```

>[!quote]- 通俗易懂的理解
>- 可以把 NDF 看成“无数小镜子朝向的统计直方图”。
>- 直方图不同，高光就会不同：有的尖、有的宽、有的拖尾长。
>- 所以 NDF 选型本质是在选“材质高光性格”。

>[!example]- 面试回答简版模板
>`NDF是微表面法线分布函数，对应Cook-Torrance的D项，决定高光形态。常见有GGX、Beckmann、Blinn-Phong，实时PBR里GGX最常用，因为其粗糙表面的高光尾部更自然且工程配套成熟。`
