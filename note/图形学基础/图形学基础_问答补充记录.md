### 记录 1
- 日期：2026-03-13
- 题目：光照模型计算（Phong/Blinn/PBR/BRDF）、法线贴图、阴影查询、环境反射、透明度处理是在干什么？
- 补充知识点：
  - `Phong / Blinn-Phong / PBR / BRDF`：都是“片元着色阶段如何算光”的规则。
  - `Phong`：经验模型，计算漫反射+镜面反射，简单但物理一致性一般。
  - `Blinn-Phong`：用半程向量改进高光方向计算，稳定性比 Phong 更好。
  - `PBR`：基于物理的材质与光照工作流（常见 metallic-roughness），跨光照条件更稳定。
  - `BRDF`：双向反射分布函数，是“入射光和观察方向决定反射强度”的函数框架；PBR 通常基于某种 BRDF（如 Cook-Torrance）。
  - `法线贴图（Normal Map）`：不增加模型面数，仅改变每像素法线方向，让平面呈现凹凸细节。
  - `阴影查询（Shadow Query）`：判断像素是否被光源遮挡（例如 shadow map 深度比较），决定直射光是否被衰减。
  - `环境反射（Environment Reflection）`：使用环境贴图/IBL 模拟对周围环境的反射，粗糙度决定反射清晰度。
  - `透明度处理（Alpha/Blend）`：让当前像素与背景按透明度合成，不是简单覆盖；常依赖排序（尤其半透明）。
- 通俗解释：
  - 光照模型：决定“这个像素该多亮、什么颜色、有没有高光”。
  - 法线贴图：决定“看起来凹凸不平，但其实模型没变复杂”。
  - 阴影查询：决定“这个点是不是被挡住了，见不见得到光”。
  - 环境反射：决定“会不会把周围环境映在表面上”。
  - 透明度处理：决定“这个像素和后景怎么叠色”。
- 面试可复述模板：
  - 光照模型负责定义像素光照计算规则；法线贴图提供微表面方向细节；阴影查询决定可见光贡献；环境反射补充间接环境光响应；透明度处理负责与背景做 alpha 混合。它们共同决定片元阶段的最终视觉效果。

### 记录 2
- 日期：2026-03-13
- 题目：延迟渲染在移动端的劣势为什么明显？
- 补充知识点：
  - 关键矛盾：移动端最敏感的是外部内存带宽、功耗和热预算；而延迟渲染的 G-Buffer 机制恰好会增加读写压力。
  - `G-Buffer 多目标写入 + 光照阶段再次读取`：导致外存访问显著增加，常成为移动端瓶颈。
  - `TBDR（Tile-Based Deferred Rendering）` 硬件背景：移动 GPU 依赖片上 tile 缓冲减少外存访问；胖 G-Buffer 会削弱这一优势。
  - `功耗与热`：带宽开销上升会抬高功耗，长时间运行易触发热降频（thermal throttling），导致帧率不稳。
  - `MSAA 成本`：延迟渲染与 MSAA 组合通常代价更高，实际项目常用 TAA/FXAA 或降低采样数折中。
  - `透明处理复杂`：透明常需单独 forward pass，形成混合管线并增加工程复杂度与额外 pass 成本。
  - `结论`：不是延迟渲染不能用，而是移动端上“收益与代价”常不如 Forward/Forward+。
- 通俗解释：
  - 移动端像“电池和散热都很紧的轻薄本”，最怕内存来回搬大数据。
  - 延迟渲染像“先把每个像素档案记很全，再反复查档案算光”，算得清楚但搬运太重。
  - 结果往往是：一开始看起来没问题，跑久了发热后掉频、掉帧。
- 面试可复述模板：
  - 延迟渲染在移动端的主要劣势是 G-Buffer 的写入与读取造成高带宽开销，进而带来功耗和热压力；再加上 MSAA 与透明处理复杂度，整体成本偏高。实际项目常采用 Forward/Forward+ 或混合管线，以换取更稳定的功耗和帧率表现。

### 记录 3
- 日期：2026-03-14
- 题目：为什么 `w` 在变换过程中会不是 `1`？
- 补充知识点：
  - 顶点初始写成齐次坐标时，点通常是 `p_local = (x, y, z, 1)`，这里的 `w=1` 表示“这是一个点”。
  - 在模型/观察等仿射变换（`M`、`V`）里，矩阵最后一行常为 `[0, 0, 0, 1]`，因此 `w` 保持不变。
  - 到投影变换（`P`）时，最后一行通常不再是 `[0, 0, 0, 1]`，会让 `w_clip` 与深度（`z_view`）相关。
  - 透视投影的关键就是让 `w` 编码距离信息，再做 `ndc = clip.xyz / clip.w` 得到远小近大的视觉效果。
  - 因此：`w` 不是“永远等于 1 的标志位”，而是齐次坐标体系里的有效计算量，尤其在透视阶段非常关键。
  - 额外区分：方向向量常写 `w=0`，这会使其不受平移影响（常用于法线/方向计算语义区分）。
- 通俗解释：
  - 可以把 `w` 理解成“投影前附带的尺度因子”。
  - 前面几步（摆物体、挪相机）通常不改这个因子；到透视镜头这一步，系统会把“距离信息”写进 `w`。
  - 后面一除 `w`，远处点被除得更小，于是你看到真实的透视效果。
- 面试可复述模板：
  - 初始点的齐次坐标通常是 `w=1`，在 `M/V` 这类仿射变换下 `w` 一般保持不变；但透视投影矩阵会让 `w_clip` 与视空间深度相关，随后通过 `x/w,y/w,z/w` 完成透视除法，形成远小近大。因此 `w` 在投影后不一定是 1，这是透视成像所必需的。
