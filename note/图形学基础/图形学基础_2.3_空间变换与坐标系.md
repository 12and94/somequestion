## 题目：模型到屏幕的空间变换过程？有哪些坐标空间？
>[!NOTE]- 标准准确的说法
>- **完整坐标空间链路（顶点阶段到光栅化前）**
>- 1) 模型空间（Model/Object）：顶点在“物体自身局部坐标系”中的位置。
>- 2) 世界空间（World）：把多个物体统一放进同一场景坐标系。
>- 3) 观察空间（View/Camera/Eye）：以相机为原点、朝向为基的坐标系。
>- 4) 裁剪空间（Clip）：乘投影矩阵后的齐次坐标，用于裁剪。
>- 5) NDC（Normalized Device Coordinates）：`clip.xyz / clip.w` 透视除法后的标准化空间。
>- 6) 视口/屏幕空间（Viewport/Screen）：把 NDC 映射到像素坐标与深度缓冲范围。
>- **常见扩展空间（面试常问）**
>- 切线空间（Tangent）：法线贴图常在该空间存储，着色时再转到世界/观察空间。
>- 纹理空间（UV）：2D 采样坐标，不参与几何变换链，但与材质采样强相关。
>- 阴影空间（Light Clip/NDC）：做 Shadow Map 时，顶点还会变换到“光源视角”空间。
>- **数学表达（列向量约定）**
>- `p_world = M * p_local`
>- `p_view  = V * p_world`
>- `p_clip  = P * p_view`
>- `p_ndc   = p_clip.xyz / p_clip.w`
>- `p_screen = ViewportMap(p_ndc)`
>- 合并写法：`p_clip = P * V * M * p_local`。
>- **视口变换（示意）**
>- `x_screen = (x_ndc * 0.5 + 0.5) * width  + x_min`
>- `y_screen = (y_ndc * 0.5 + 0.5) * height + y_min`
>- `z_screen = z_ndc * (z_max - z_min) + z_min`（具体随 API 深度约定变化）
>- **裁剪判定（齐次裁剪）**
>- OpenGL 传统约定：`-w<=x<=w, -w<=y<=w, -w<=z<=w`。
>- D3D/Vulkan 常见约定：`-w<=x<=w, -w<=y<=w, 0<=z<=w`。
>- 这是“在 Clip 空间先裁剪，再做 /w”的核心原因。
>- **为什么必须分这么多空间**
>- 模型空间便于复用模型资产。
>- 世界空间便于场景管理（碰撞、光照、剔除）。
>- 观察空间便于以相机为中心做可见性与投影。
>- NDC/视口空间便于硬件统一光栅化和分辨率映射。
>- **高频工程细节（加分点）**
>- 1) API 差异：OpenGL 与 D3D/Vulkan 的 NDC z 范围不同。
>- 2) 屏幕 Y 轴方向可能因 API/RT 约定出现翻转。
>- 3) 深度是非线性的（透视投影下近处精度高、远处精度低）。
>- 4) 大场景常用 Reverse-Z + 浮点深度提升远距离精度。
>- 5) 做 SSAO/SSR 等屏幕后处理，常需从深度反推 View 空间位置。

>[!tip]- 代码示例
>```mermaid
>flowchart LR
>  A[Model/Object] -->|M| B[World]
>  B -->|V| C[View/Camera]
>  C -->|P| D[Clip (x,y,z,w)]
>  D -->|/w| E[NDC]
>  E -->|Viewport| F[Screen/Depth]
>```
>
>```cpp
>// 列向量约定（HLSL/GLSL 伪代码思想）
>float4 pLocal = float4(pos, 1.0);
>float4 pWorld = mul(M, pLocal);
>float4 pView  = mul(V, pWorld);
>float4 pClip  = mul(P, pView);
>
>// 透视除法
>float3 pNdc = pClip.xyz / pClip.w;
>
>// NDC -> Viewport
>float2 pScreen;
>pScreen.x = (pNdc.x * 0.5f + 0.5f) * viewportWidth  + viewportX;
>pScreen.y = (pNdc.y * 0.5f + 0.5f) * viewportHeight + viewportY;
>
>// 深度写入（示意，实际受 API 深度约定影响）
>float depth = pNdc.z;
>```

>[!quote]- 通俗易懂的理解
>- 可以把它理解为“从演员到电影银幕”的流水线：
>- 模型空间是演员自己的动作坐标，世界空间是片场，观察空间是摄像机看到的世界。
>- 投影后进入裁剪与标准化（Clip/NDC），最后贴到具体像素网格上（Viewport/Screen）。
>- 分层的本质是“把复杂问题拆成可复用、可维护、可硬件加速的固定阶段”。

>[!example]- 面试回答简版模板
>`模型到屏幕通常是 Object -> World -> View -> Clip -> NDC -> Screen。数学上是 p_clip=P*V*M*p_local，随后做 p_ndc=p_clip.xyz/p_clip.w，再经 viewport 映射到像素与深度。Clip 空间先做齐次裁剪，NDC 范围和深度约定受 API 影响（如 OpenGL z∈[-1,1]，D3D/Vulkan 常见 z∈[0,1]）。工程上要注意深度非线性、Y 翻转和 Reverse-Z 等细节。`

## 题目：MVP矩阵各自的作用？如何推导？
>[!NOTE]- 标准准确的说法
>- **M（Model）矩阵**
>- 把局部顶点放到世界空间，编码平移/旋转/缩放（TRS）。
>- **V（View）矩阵**
>- 把世界坐标变到相机坐标，本质是“相机位姿矩阵的逆”。
>- **P（Projection）矩阵**
>- 把观察空间映射到裁剪空间，定义视锥体（透视）或盒体（正交）。
>- **推导思路**
>- 从“点在不同坐标系表达”的线性代数关系出发，连续应用基变换。
>- 最终得到：`p_clip = P * V * M * p_local`（列向量约定）。
>- **常见易错点**
>- 乘法顺序与行/列向量约定强相关。
>- API 左手/右手坐标与深度范围约定会影响具体矩阵形式。

>[!tip]- 代码示例
>```cpp
>// 伪代码
>mat4 M = Translate(T) * Rotate(R) * Scale(S);
>mat4 V = Inverse(CameraWorldTransform);
>mat4 P = Perspective(fovY, aspect, near, far);
>
>float4 clip = P * V * M * float4(localPos, 1.0);
>```

>[!quote]- 通俗易懂的理解
>- M 是“把物体摆到场景里”，V 是“把场景换成相机眼里的坐标”，P 是“把 3D 透视压到屏幕可投影形态”。

>[!example]- 面试回答简版模板
>`MVP 分工清晰：M 决定物体姿态，V 决定观察视角，P 决定投影规则。推导本质是坐标系连续变换，最终得到 p_clip = P*V*M*p_local。`

## 题目：齐次坐标是什么？有什么好处？
>[!NOTE]- 标准准确的说法
>- **定义**
>- 在 3D 坐标后补一维 `w`，把点/向量写成 4 维形式。
>- 点：`(x, y, z, 1)`；方向向量：`(x, y, z, 0)`。
>- **好处**
>- 1) 把平移纳入统一矩阵乘法（4x4），与旋转缩放统一表达。
>- 2) 方便组合变换（多个矩阵可连乘）。
>- 3) 支持投影与透视除法流程（Clip -> NDC）。
>- **几何含义**
>- 齐次坐标按比例等价，如 `(2x,2y,2z,2w)` 与 `(x,y,z,w)` 表示同一点（`w!=0`）。

>[!tip]- 代码示例
>```cpp
>float4 point     = float4(x, y, z, 1.0); // 点
>float4 direction = float4(dx,dy,dz,0.0); // 方向
>
>// 平移只影响 w=1 的点，不影响方向向量
>float4 p2 = T * point;
>float4 d2 = T * direction;
>```

>[!quote]- 通俗易懂的理解
>- 给坐标加上 `w` 后，图形学把“平移、旋转、缩放、投影”都变成同一种矩阵运算，像统一接口一样好组合。

>[!example]- 面试回答简版模板
>`齐次坐标通过增加 w 把平移矩阵化，统一仿射与投影变换，并为透视除法提供数学基础。点通常 w=1，方向向量 w=0。`

## 题目：NDC是什么？坐标范围？
>[!NOTE]- 标准准确的说法
>- **NDC 是什么**
>- NDC（Normalized Device Coordinates）是“裁剪空间坐标做透视除法后”的标准化坐标。
>- 即：`p_ndc = p_clip.xyz / p_clip.w`。
>- 它处在 `Clip -> NDC -> Viewport` 这个阶段中间，是光栅化前的关键空间。
>- **为什么要有 NDC**
>- 把不同相机参数、不同分辨率下的几何结果统一到同一个标准范围，便于 GPU 后续固定流程处理。
>- **范围（面试必答）**
>- `x, y`：通常都在 `[-1, 1]`。
>- `z`：依 API 约定不同：
>- OpenGL 传统常见 `[-1, 1]`。
>- D3D / Vulkan 常见 `[0, 1]`。
>- **和 Clip / Screen 的关系**
>- Clip 是齐次坐标（`x,y,z,w`），可做齐次裁剪。
>- NDC 是做了 `/w` 后的三维标准坐标（`x,y,z`）。
>- Screen 是 NDC 经过视口映射后的像素坐标与深度值。
>- **一句话定位**
>- NDC 不是世界坐标，也不是屏幕像素坐标；它是两者之间的“标准化中间层”。

>[!tip]- 代码示例
>```cpp
>// 1) 先得到 Clip 坐标
>float4 clip = mul(P, mul(V, mul(M, float4(localPos, 1.0f))));
>
>// 2) 透视除法 -> NDC
>float3 ndc;
>ndc.x = clip.x / clip.w;
>ndc.y = clip.y / clip.w;
>ndc.z = clip.z / clip.w;
>
>// 3) NDC -> 屏幕像素（示意）
>float2 screen;
>screen.x = (ndc.x * 0.5f + 0.5f) * width  + xMin;
>screen.y = (ndc.y * 0.5f + 0.5f) * height + yMin;
>```
>
>```text
>数值例子：clip=(2, 1, 5, 10)
>NDC = (2/10, 1/10, 5/10) = (0.2, 0.1, 0.5)
>```
>
>```mermaid
>flowchart LR
>  A[Clip x,y,z,w] --> B[Perspective Divide /w]
>  B --> C[NDC x,y,z]
>  C --> D[Viewport Mapping]
>  D --> E[Screen Pixels + Depth]
>```

>[!quote]- 通俗易懂的理解
>- 可以把 NDC 理解成“统一坐标盒子”：
>- 不管你相机 FOV 多大、分辨率多高，先都映射进同一个标准盒子里。
>- 这样后面再贴到 1080p、2K、4K 屏幕时，流程都一致。
>- 你看到的“远小近大”本质来自 `/w` 这一步，而这一步的结果正是 NDC。

>[!example]- 面试回答简版模板
>`NDC 是 clip 坐标做透视除法（x/w,y/w,z/w）后的标准化空间，位于投影与视口映射之间。x/y 通常在 [-1,1]，z 依 API 而异：OpenGL 常见 [-1,1]，D3D/Vulkan 常见 [0,1]。NDC 的作用是把结果统一到标准范围，便于后续光栅化和屏幕映射。`

>[!info]- 对应示例
>- `note/图形学基础/图形学基础_2.3_空间变换与坐标系.md`

## 题目：透视投影和正交投影的区别？
>[!NOTE]- 标准准确的说法
>- **透视投影（Perspective）**
>- 保留远小近大，符合人眼视觉。
>- 平行线在投影后可能汇聚。
>- 适合主摄像机、真实空间表现。
>- **正交投影（Orthographic）**
>- 不体现远近缩放，尺度与距离无关。
>- 平行关系保持。
>- 适合 UI、CAD、编辑器、技术可视化。
>- **本质差异**
>- 透视依赖 `w` 产生非线性深度/尺度变化；正交通常是线性映射。

>[!tip]- 代码示例
>```cpp
>mat4 P_persp = Perspective(fovY, aspect, near, far);
>mat4 P_ortho = Ortho(left, right, bottom, top, near, far);
>```

>[!quote]- 通俗易懂的理解
>- 透视像手机拍照，远处会变小；正交像工程图纸，尺寸不会因远近而变化。

>[!example]- 面试回答简版模板
>`透视投影强调真实视觉（远小近大），正交投影强调尺寸一致性。游戏主视图多用透视，UI/CAD 等场景常用正交。`

## 题目：视口变换的作用？
>[!NOTE]- 标准准确的说法
>- **定义**
>- 视口变换把 NDC 映射到具体渲染目标区域（像素坐标 + 深度区间）。
>- **核心公式（示意）**
>- `x_screen = (x_ndc * 0.5 + 0.5) * width + x_offset`
>- `y_screen = (y_ndc * 0.5 + 0.5) * height + y_offset`
>- `z_screen = map(z_ndc, depthRange)`
>- **用途**
>- 决定画面落在屏幕哪一块（全屏、分屏、小窗、后处理目标）。

>[!tip]- 代码示例
>```cpp
>// 伪代码
>SetViewport(x, y, width, height);
>SetDepthRange(zMin, zMax);
>```

>[!quote]- 通俗易懂的理解
>- NDC 是“标准地图坐标”，视口变换是把这张地图贴到具体屏幕区域上。

>[!example]- 面试回答简版模板
>`视口变换负责把 NDC 坐标映射到具体像素区域和深度范围，是从标准化空间走向实际屏幕坐标的最后一步。`

## 题目：切线空间是什么？
>[!NOTE]- 标准准确的说法
>- **定义**
>- 切线空间是贴在模型表面的局部坐标系，由 `T`（切线）、`B`（副切线/bitangent）、`N`（法线）构成 TBN。
>- **用途**
>- 法线贴图通常存的是“切线空间法线”，需要通过 TBN 转到世界/观察空间参与光照。
>- **关键质量点**
>- TBN 构建不一致会导致接缝、翻转、光照跳变。
>- 资产烘焙与运行时 TBN 约定需一致（常见 MikkTSpace 体系）。

>[!tip]- 代码示例
>```glsl
>mat3 TBN = mat3(normalize(T), normalize(B), normalize(N));
>vec3 n_tan = texture(normalMap, uv).xyz * 2.0 - 1.0;
>vec3 n_ws  = normalize(TBN * n_tan);
>```

>[!quote]- 通俗易懂的理解
>- 切线空间就是“贴在表面上的小罗盘”。
>- 法线贴图告诉你小罗盘里指针朝哪，TBN 再把这个方向翻译到世界坐标。

>[!example]- 面试回答简版模板
>`切线空间是表面局部坐标系（TBN），用于表达法线贴图方向；渲染时通过 TBN 把切线空间法线转换到统一光照空间。`

## 题目：裁剪空间做了什么？
>[!NOTE]- 标准准确的说法
>- **位置**
>- 顶点经投影矩阵后进入裁剪空间（Clip Space）。
>- **功能**
>- 对视锥体外图元做裁剪/剔除，避免后续无效光栅化。
>- 通过后再做透视除法得到 NDC。
>- **意义**
>- 这是“可见性几何筛选”的关键环节，减少后续像素开销。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[Projection后进入Clip] --> B{在裁剪体内?}
>  B -->|完全在外| C[剔除]
>  B -->|部分相交| D[裁剪重建图元]
>  B -->|在内| E[直接通过]
>  D --> E
>  E --> F[透视除法->NDC]
>```

>[!quote]- 通俗易懂的理解
>- 裁剪空间像“电影院检票口”：不在可见区域内的图元先拦下，减少后续算力浪费。

>[!example]- 面试回答简版模板
>`裁剪空间是投影后的中间空间，用于做视锥裁剪。通过裁剪后才做透视除法进入 NDC，从而减少无效光栅化开销。`

## 题目：齐次空间裁剪怎么做？
>[!NOTE]- 标准准确的说法
>- **判定基础**
>- 在齐次裁剪空间，常用与 `w` 比较的边界条件判断可见性（具体边界细节受 API 约定影响）。
>- **三类结果**
>- 完全在内：直接通过。
>- 完全在外：整图元剔除。
>- 部分相交：在裁剪平面求交，重建裁剪后图元。
>- **算法思路**
>- 本质是多平面裁剪；经典可参考 Sutherland-Hodgman 思路。
>- **为什么在齐次空间做**
>- 可在透视除法前保持线性插值关系，避免先除法导致的几何不一致问题。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[图元进入齐次裁剪空间] --> B{全部在内?}
>  B -->|是| C[通过]
>  B -->|否| D{全部在外?}
>  D -->|是| E[剔除]
>  D -->|否| F[与裁剪面求交并重建图元]
>  F --> C
>```
>
>```cpp
>// 伪代码（概念）
>for each clipPlane {
>    polygon = ClipPolygonAgainstPlane(polygon, clipPlane);
>    if (polygon.empty()) break;
>}
>```

>[!quote]- 通俗易懂的理解
>- 图元跨出可见边界时，不是整块扔掉，而是把越界部分“切掉”，只保留镜头内那部分继续渲染。

>[!example]- 面试回答简版模板
>`齐次裁剪通过与 w 相关的边界关系判断可见性；跨界图元在裁剪平面上求交并重建，再进入透视除法和后续光栅化。`
