## 题目：如何判断点在三角形内？
>[!NOTE]- 标准准确的说法（详细版）
>- **先说结论**
>- 最常用有三类：
>- 1. 重心坐标法（最通用，也能顺便做插值）。
>- 2. 边函数/同向叉积法（栅格化里常用，速度快）。
>- 3. 面积法（直观但数值稳定性相对差一些）。
>- 
>- **方法1：重心坐标法（推荐你优先掌握）**
>- 对三角形 `A,B,C` 和点 `P`，若存在：
> $$
> P = uA + vB + wC,\quad u+v+w=1
> $$
>- 且 `u,v,w >= 0`，则 `P` 在三角形内部（含边界）。
>- 若某个权重 `< 0`，则在外部。
>- 
>- **方法2：边函数/同向叉积法**
>- 对三条边分别算“点在边哪一侧”。
>- 若三次符号一致（都非负或都非正），点在内。
>- 
>- 可写成边函数：
> $$
> E_{ab}(p) = (p_x-a_x)(b_y-a_y) - (p_y-a_y)(b_x-a_x)
> $$
>- 对三条边 `AB/BC/CA` 的 `E` 符号一致即可。
>- 
>- **方法3：面积法（直觉强）**
>- 计算 `Area(PAB)+Area(PBC)+Area(PCA)` 是否约等于 `Area(ABC)`。
>- 需加 epsilon，避免浮点误差。
>- 
>- **工程注意点**
>- 1. 一定加 `epsilon`，别直接用 `==`。
>- 2. 退化三角形（面积接近 0）要提前判掉。
>- 3. 若后续还要做 UV/法线插值，重心法最省事。

>[!tip]- 代码示例
>```cpp
>// 重心坐标 inside test（简化）
>bool PointInTri(float2 p, float2 a, float2 b, float2 c)
>{
>    float2 v0 = b - a, v1 = c - a, v2 = p - a;
>    float d00 = dot(v0, v0);
>    float d01 = dot(v0, v1);
>    float d11 = dot(v1, v1);
>    float d20 = dot(v2, v0);
>    float d21 = dot(v2, v1);
>
>    float denom = d00 * d11 - d01 * d01;
>    if (abs(denom) < 1e-8f) return false; // 退化
>
>    float v = (d11 * d20 - d01 * d21) / denom;
>    float w = (d00 * d21 - d01 * d20) / denom;
>    float u = 1.0f - v - w;
>
>    const float eps = -1e-6f;
>    return (u >= eps && v >= eps && w >= eps);
>}
>```

>[!quote]- 通俗易懂的理解
>- 你可以把三角形当成“由 A/B/C 三个点按比例混出来的区域”。
>- 比例都不为负，就说明点没跑出三角形外。

>[!example]- 面试回答简版模板
>`点在三角形内判断我常用重心坐标：求出u/v/w后判断是否都>=0且和为1。它不仅能做inside test，还能直接用于UV和法线插值。栅格化里也常用边函数做同侧判定。`

## 题目：已知法线和入射向量求反射向量？
>[!NOTE]- 标准准确的说法（详细版）
>- **公式先背住**
>- 入射向量 `I`（指向表面），单位法线 `N`，反射向量：
> $$
> R = I - 2(I\cdot N)N
> $$
>- 
>- **这个公式怎么来的（直觉）**
>- 把 `I` 分解成：
>- 平行法线分量 + 垂直法线分量。
>- 镜面反射相当于“法线方向分量取反，切向分量不变”。
>- 代数化简后就是上面公式。
>- 
>- **使用前提**
>- `N` 必须归一化。
>- `I` 最好也归一化（至少方向要正确），否则反射方向偏差大。
>- 
>- **常见场景**
>- 镜面高光方向计算。
>- 环境反射采样方向。
>- SSR/反射探针方向。

>[!tip]- 代码示例
>```cpp
>float3 Reflect(float3 I, float3 N)
>{
>    N = normalize(N);
>    I = normalize(I);
>    return I - 2.0f * dot(I, N) * N;
>}
>```

>[!quote]- 通俗易懂的理解
>- 就是把入射向量沿法线“镜像翻折”过去。

>[!example]- 面试回答简版模板
>`反射方向公式是R=I-2*(I·N)*N，本质是把I在法线方向上的分量取反。工程里要先归一化法线，常用于镜面反射和环境采样方向。`

## 题目：射线与球体/AABB/三角形求交？
>[!NOTE]- 标准准确的说法（详细版）
>- **统一起点**
>- 射线参数方程：
> $$
> P(t)=O+tD,\quad t\ge 0
> $$
>- 所有求交都在问：是否存在满足几何约束的 `t`，并取最小正 `t`。
>- 
>- **1）射线-球体**
>- 球：
> $$
> ||P-C||^2=r^2
> $$
>- 代入 `P(t)` 得一元二次：
> $$
> at^2+bt+c=0
> $$
>- 判别式：
> $$
> \Delta=b^2-4ac
> $$
>- `\Delta<0` 无交；`=0` 相切；`>0` 两个交点。
>- 取最小正根作为命中点。
>- 
>- **2）射线-AABB（slab 法）**
>- 在 x/y/z 三个轴上分别求进入和离开区间：
> $$
> t_{min} = \max(t_{x1},t_{y1},t_{z1}),\quad
> t_{max} = \min(t_{x2},t_{y2},t_{z2})
> $$
>- 若 `tmax < tmin` 则无交。
>- 这是 BVH 遍历最常用的粗测。
>- 
>- **3）射线-三角形（Moller-Trumbore）**
>- 直接解 `t,u,v`：
>- `t` 是距离参数，`u,v` 是重心坐标。
>- 命中条件：
>- `t >= 0`
>- `u >= 0`
>- `v >= 0`
>- `u+v <= 1`
>- 该方法无需先求平面方程，效率高，实时和光追都常用。
>- 
>- **工程流程（最常见）**
>- `Ray -> BVH node(AABB) 粗测 -> Triangle 精测 -> 取最近t`。
>- 
>- **常见坑**
>- 平行/近似平行时分母接近 0，要加 epsilon。
>- 射线方向分量为 0 时 slab 需特殊处理。
>- 求最近交时别忘记维护当前 `tHit` 上界，便于 early-out。

>[!tip]- 代码示例
>```mermaid
>flowchart LR
>  A[Ray O,D] --> B[AABB Slab Test]
>  B -->|命中| C[Triangle MT Test]
>  C --> D{t更小?}
>  D -->|是| E[更新最近命中]
>  D -->|否| F[忽略]
>  B -->|未命中| G[剔除BVH节点]
>```

>[!quote]- 通俗易懂的理解
>- 先用便宜的 AABB 快速排除一大批，再对少量候选三角形做精确求交。

>[!example]- 面试回答简版模板
>`射线求交统一用P(t)=O+tD。球体用二次方程判别式，AABB用slab区间相交，三角形常用Moller-Trumbore解t/u/v。工程里是BVH+AABB粗筛后再三角形精测。`

## 题目：欧拉角有什么问题？四元数如何解决？
>[!NOTE]- 标准准确的说法（详细版）
>- **欧拉角的问题不是“不能用”，而是有边界**
>- 欧拉角是按固定顺序绕三个轴旋转（如 yaw-pitch-roll）。
>- 
>- **主要问题 1：万向节锁（Gimbal Lock）**
>- 当中间轴旋转到某些角度（典型 `pitch=+-90°`）时，两个旋转轴重合，自由度丢失。
>- 结果：你发现“某个方向转不动了”。
>- 
>- **主要问题 2：插值不自然**
>- 对欧拉角分量直接线性插值，会出现：
>- 旋转速度不均匀。
>- 路径绕远（不是最短旋转路径）。
>- 
>- **四元数怎么解决**
>- 用单位四元数表示旋转，避免欧拉角轴重合问题。
>- 旋转组合用四元数乘法，稳定且数值表现好。
>- 插值可用 slerp（球面线性插值），旋转路径更自然。
>- 
>- **slerp 形式（示意）**
> $$
> q(t)=\frac{\sin((1-t)\theta)}{\sin\theta}q_0 + \frac{\sin(t\theta)}{\sin\theta}q_1
> $$
>- 其中 `theta` 是两四元数夹角。
>- 
>- **工程实践（很重要）**
>- 编辑器/UI 仍常用欧拉角（人类更好理解）。
>- 运行时和动画混合多用四元数。
>- 每次运算后常要 normalize，避免误差积累。

>[!tip]- 代码示例
>```cpp
>// 欧拉角输入 -> 四元数运行时
>quat q = QuatFromEuler(yaw, pitch, roll);
>quat qBlend = normalize(slerp(q0, q1, t));
>mat3 R = Mat3FromQuat(qBlend);
>```

>[!quote]- 通俗易懂的理解
>- 欧拉角适合“输入和展示”，四元数适合“计算和插值”。

>[!example]- 面试回答简版模板
>`欧拉角直观但有万向节锁和插值路径问题。四元数用单位球面表示旋转，组合稳定，并可通过slerp得到平滑且接近最短路径的插值。工程里常是欧拉输入、四元数计算。`

## 题目：点到线段/三角形的距离？
>[!NOTE]- 标准准确的说法（详细版）
>- **统一套路：先找最近点，再算长度**
>- 
>- **1）点到线段距离**
>- 线段 `AB`，点 `P`：
>- 先算投影参数
> $$
> t = \frac{(P-A)\cdot(B-A)}{||B-A||^2}
> $$
>- 再 `t=clamp(t,0,1)`。
>- 最近点
> $$
> Q=A+t(B-A)
> $$
>- 距离 `d=||P-Q||`。
>- 
>- **2）点到三角形距离**
>- 步骤：
>- 1. 将点投影到三角形平面，得 `P'`。
>- 2. 若 `P'` 在三角形内，距离是点到平面的垂距。
>- 3. 若不在内，距离是 `min(dist(P,AB), dist(P,BC), dist(P,CA))`。
>- 
>- **工程应用**
>- 碰撞检测。
>- SDF 生成。
>- 最近表面点查询。
>- 
>- **常见坑**
>- 线段退化为点（`A==B`）要单独处理。
>- 三角形退化时需退化到线段/点距离。

>[!tip]- 代码示例
>```cpp
>float DistPointSegment(float3 P, float3 A, float3 B)
>{
>    float3 AB = B - A;
>    float denom = dot(AB, AB);
>    if (denom < 1e-8f) return length(P - A);
>
>    float t = dot(P - A, AB) / denom;
>    t = clamp(t, 0.0f, 1.0f);
>    float3 Q = A + t * AB;
>    return length(P - Q);
>}
>```

>[!quote]- 通俗易懂的理解
>- 距离题的核心不是“公式背多少”，而是“最近点在哪”。

>[!example]- 面试回答简版模板
>`点到线段先做投影再clamp到[0,1]，得到最近点后求长度。点到三角形先看投影点是否在内，在内取平面垂距，不在内取三条边线段距离最小值。`

## 题目：重心坐标怎么计算？
>[!NOTE]- 标准准确的说法（详细版）
>- **定义**
>- 对三角形 `A,B,C` 与点 `P`：
> $$
> P = uA + vB + wC,\quad u+v+w=1
> $$
>- `u,v,w` 就是重心坐标。
>- 
>- **计算法1：点积法（工程常用）**
>- 令
>- `v0 = B-A, v1 = C-A, v2 = P-A`。
>- 则
> $$
> v = \frac{d_{11}d_{20}-d_{01}d_{21}}{d_{00}d_{11}-d_{01}^2},
> \quad
> w = \frac{d_{00}d_{21}-d_{01}d_{20}}{d_{00}d_{11}-d_{01}^2},
> \quad
> u = 1-v-w
> $$
>- 其中 `dij` 是相应点积。
>- 
>- **计算法2：面积比法（直观）**
> $$
> u=\frac{Area(PBC)}{Area(ABC)},
> v=\frac{Area(PCA)}{Area(ABC)},
> w=\frac{Area(PAB)}{Area(ABC)}
> $$
>- 
>- **为什么这么重要**
>- inside test：`u,v,w>=0`。
>- 属性插值：
> $$
> attr_P = u\,attr_A + v\,attr_B + w\,attr_C
> $$
>- 光栅化中 UV/法线/颜色都靠它插值。

>[!tip]- 代码示例
>```cpp
>// 已知u,v,w后插值属性
>float2 uvP = u * uvA + v * uvB + w * uvC;
>float3 nP  = normalize(u * nA + v * nB + w * nC);
>```

>[!quote]- 通俗易懂的理解
>- 重心坐标就是“这个点由 A/B/C 各贡献了多少比例”。

>[!example]- 面试回答简版模板
>`重心坐标把点表示成三个顶点的线性组合，权重和为1。它既能判断点是否在三角形内，也能做属性插值，是光栅化和几何计算的基础。`

## 题目：透视矫正插值的原理？
>[!NOTE]- 标准准确的说法（详细版）
>- **为什么会失真**
>- 在透视投影下，屏幕空间不是“世界空间线性保持”。
>- 直接对属性做屏幕线性插值会导致纹理拉伸/漂移。
>- 
>- **正确方法（核心公式）**
>- 先插值 `attr/w` 和 `1/w`，最后恢复：
> $$
> attr = \frac{\text{interp}(attr/w)}{\text{interp}(1/w)}
> $$
>- 这就是 perspective-correct interpolation。
>- 
>- **更完整写法（按重心）**
> $$
> attr_P = \frac{u\frac{attr_A}{w_A}+v\frac{attr_B}{w_B}+w\frac{attr_C}{w_C}}
> {u\frac{1}{w_A}+v\frac{1}{w_B}+w\frac{1}{w_C}}
> $$
>- 
>- **为什么有效**
>- 齐次坐标里，投影前量在 clip 空间保持线性。
>- 通过“先除后插再还原”恢复正确透视关系。
>- 
>- **工程表现**
>- 没做透视矫正：斜视角下纹理会“游走”。
>- 做了透视矫正：贴图在透视下依然稳定。

>[!tip]- 代码示例
>```cpp
>// 概念伪代码
>float invW = Interp(1.0 / w0, 1.0 / w1, 1.0 / w2, bary);
>float2 uvOverW = Interp(uv0 / w0, uv1 / w1, uv2 / w2, bary);
>float2 uv = uvOverW / invW;
>```

>[!quote]- 通俗易懂的理解
>- 先把属性“去透视”再插值，最后“还原透视”，才能不变形。

>[!example]- 面试回答简版模板
>`透视投影下屏幕线性插值不成立，正确做法是插值attr/w和1/w再相除恢复属性。这能避免纹理在远近变化下出现透视失真。`

## 题目：凸包算法？
>[!NOTE]- 标准准确的说法（详细版）
>- **先定义**
>- 凸包是“包住所有点的最小凸边界”。
>- 2D 是最小凸多边形，3D 是最小凸多面体。
>- 
>- **2D 常见算法**
>- **Monotone Chain（工程常用）**
>- 1. 点按 `(x,y)` 排序。
>- 2. 扫一遍构下壳，遇到非左转就弹栈。
>- 3. 反向扫构上壳。
>- 4. 拼接上下壳得凸包。
>- 复杂度 `O(n log n)`（排序主导）。
>- 
>- **Graham Scan**
>- 1. 找最低点。
>- 2. 按极角排序。
>- 3. 栈维护转向。
>- 同样常见 `O(n log n)`。
>- 
>- **转向判定（核心）**
>- 对三点 `A,B,C`：
> $$
> cross(B-A, C-A)
> $$
>- `>0` 左转，`<0` 右转，`=0` 共线。
>- 
>- **工程应用**
>- 碰撞包围。
>- 点集外轮廓提取。
>- 路径规划与几何预处理。

>[!tip]- 代码示例
>```cpp
>// Monotone Chain核心思路（伪）
>sort(points.begin(), points.end());
>for p in points:       // build lower
>  while lower.size()>=2 && cross(lower[-1]-lower[-2], p-lower[-1]) <= 0:
>    lower.pop_back();
>  lower.push_back(p);
>// upper 同理，最后拼接
>```

>[!quote]- 通俗易懂的理解
>- 像用橡皮筋把点全包住，最后橡皮筋贴着的那圈边就是凸包。

>[!example]- 面试回答简版模板
>`2D凸包常用Monotone Chain：排序后用栈维护转向，非左转就回退，最终得到上下壳。复杂度O(n log n)，在碰撞和几何预处理中很常见。`

## 题目：法线矩阵是什么？
>[!NOTE]- 标准准确的说法（详细版）
>- **先说为什么需要它**
>- 顶点位置可直接乘模型矩阵 `M`。
>- 但法线不是位置向量，若有非均匀缩放，直接乘 `M` 会方向错误。
>- 
>- **正确法线变换**
>- 取模型矩阵线性部分 `M_{3x3}`，法线矩阵：
> $$
> Nmat = (M_{3x3}^{-1})^T
> $$
>- 法线变换：
> $$
> N' = normalize(Nmat\,N)
> $$
>- 
>- **为什么是逆转置（直觉）**
>- 法线本质上要保持与切平面垂直关系。
>- 顶点经 `M` 变换后，要让法线仍垂直于变换后的切向量，推导结果就是逆转置。
>- 
>- **什么时候可简化**
>- 纯旋转：`Nmat` 等于旋转矩阵本身。
>- 均匀缩放：可近似直接乘 `M_{3x3}` 后再 normalize。
>- 非均匀缩放：必须用逆转置。
>- 
>- **常见坑**
>- 忘记重新 normalize。
>- 把完整 4x4 直接乘法线（带平移）导致错误。
>- 切线空间里 TBN 基不正交，法线贴图结果发飘。

>[!tip]- 代码示例
>```cpp
>mat3 M3 = mat3(modelMatrix);
>mat3 Nmat = transpose(inverse(M3));
>float3 Nw = normalize(Nmat * Nobj);
>```

>[!quote]- 通俗易懂的理解
>- 顶点是“点”，法线是“方向约束”。
>- 点可直接变换，法线要用逆转置来纠正缩放扭曲。

>[!example]- 面试回答简版模板
>`法线矩阵是模型矩阵3x3线性部分的逆转置，用于正确变换法线并保持几何垂直关系。非均匀缩放时必须使用，否则光照方向会明显错误。`

## 一页速记（考前30秒）
>[!NOTE]- 你最少要记住的几句话
>- 点在三角形内：`u,v,w >= 0`（重心法）。
>- 反射向量：`R = I - 2(I·N)N`。
>- 射线求交：球二次方程，AABB slab，三角形 Moller-Trumbore。
>- 欧拉角易万向节锁，四元数适合组合与 slerp。
>- 距离题统一套路：先找最近点，再算长度。
>- 透视矫正：插值 `attr/w` 与 `1/w` 再相除。
>- 凸包常见 Monotone Chain，复杂度 `O(n log n)`。
>- 法线矩阵是 `(M3x3^{-1})^T`，非均匀缩放必须用。
