## 题目：IBL的实现细节？diffuse和specular分别怎么做？
>[!NOTE]- 标准准确的说法（先白话）
>- IBL（Image Based Lighting，基于图像的光照）就是：
>- 不再只靠场景里的灯光，而是用一张“环境全景图”来模拟来自四面八方的间接光。
>- 你可以把环境图理解成一个“发光的天空盒”。
>- 
>- **为什么要分 diffuse 和 specular 两条路径？**
>- 因为它们对方向细节的敏感程度不一样：
>- 漫反射 `diffuse`：只关心“大方向上亮不亮”，属于低频信息。
>- 镜面反射 `specular`：非常关心方向和粗糙度，属于高频信息。
>- 
>- **Diffuse IBL（漫反射）流程**
>- 1. 对环境图做半球卷积，得到 `irradiance map`（辐照度贴图）。
>- 2. 运行时按法线方向 `N` 去采样辐照度。
>- 3. 再乘材质反照率 `albedo`（和能量分配系数）。
>- 
>- 对应简化公式：
> $$
> L_{diffuse} = irradiance(N) \cdot albedo \cdot k_D
> $$
>- 
>- **Specular IBL（镜面）流程**
>- 1. 先把环境图做“按粗糙度分层”的预过滤，通常写进 mip 链。
>- 2. 粗糙度越大，采样越模糊的 mip。
>- 3. 再乘一个 `BRDF LUT`（查表），补偿视角与粗糙度影响。
>- 
>- 对应简化公式：
> $$
> L_{specular} = prefilteredEnv(R, roughness) \cdot BRDF_{LUT}(N \cdot V, roughness)
> $$
>- 其中 `R` 是反射方向，`V` 是视线方向。
>- 
>- **最终合成（简化）**
> $$
> L_o \approx L_{direct} + L_{diffuse} + L_{specular} + emissive
> $$
>- 
>- **术语速记**
>- `irradiance`：单位面积接收到的总光能（偏漫反射）。
>- `prefiltered env`：按粗糙度预先模糊好的环境图。
>- `BRDF LUT`：把一部分 BRDF 积分结果预先烘好，运行时查表省算力。

>- 
>- **你刚好可能卡住的4个点（逐个讲透）**
>- **1）“按粗糙度分层”到底是什么意思？**
>- 可以理解成：同一张环境图，提前做出很多个“模糊程度不同”的版本。
>- 粗糙度小（表面光滑）就看清晰版本；粗糙度大（表面粗糙）就看模糊版本。
>- 这样运行时不用现场做高成本模糊积分，直接取对应层即可。
>- 
>- **2）mip 链是什么？**
>- mip 链就是一张纹理的多级分辨率版本：`mip0` 最清晰，`mip1` 更小，`mip2` 更小...
>- 在预过滤环境图里，常把“更高 mip”当作“更模糊反射”的近似。
>- 所以通常是：`roughness` 越大，采样的 mip level 越高（结果越糊）。
>- 
>- **3）BRDF LUT 是什么？**
>- BRDF LUT（查找表）是一张预计算 2D 纹理，输入通常是：
>- `x = N \cdot V`（法线和视线夹角）
>- `y = roughness`（粗糙度）
>- 输出是这组参数下镜面 BRDF 积分的校正系数（很多实现里是 `A` 和 `B` 两个值）。
>- 常见写法是：
> $$
> L_{specular} \approx prefilteredColor \cdot (F_0 \cdot A + B)
> $$
>- 其中 `A,B` 来自 LUT，`F_0` 是基础反射率。
>- 
>- **4）为什么还要“补偿视角和粗糙度影响”？**
>- 因为“只按 roughness 选模糊环境图”还不够。
>- 镜面强度还会被以下因素显著影响：
>- 视角（`N \cdot V`，掠射角会更亮，和 Fresnel 相关）
>- 粗糙度（会改变高光能量分布，不只是模糊程度）
>- 如果不补偿，结果常出现过亮/过暗、能量不稳定。
>- BRDF LUT 就是在补这部分“仅靠 mip 模糊无法覆盖”的项。
>- 
>- **一句话记忆**
>- `prefiltered env` 决定“反射里看见什么”（环境颜色形状）。
>- `BRDF LUT` 决定“这反射该有多亮”（角度和粗糙度校正）。

>[!tip]- 代码示例
>```mermaid
>flowchart TD
>  A[HDR 环境图] --> B[Diffuse卷积: Irradiance Map]
>  A --> C[Specular预过滤: Mip链]
>  D[BRDF积分预计算: LUT] --> E[运行时镜面采样]
>  B --> F[运行时漫反射采样]
>  C --> E
>  F --> G[合成输出]
>  E --> G
>```
>
>```cpp
>// 概念伪代码
>float3 Ld = SampleIrradiance(N) * albedo * kD;
>float3 Ls = SamplePrefilterEnv(R, roughness) * SampleBRDFLUT(NdotV, roughness);
>float3 Lo = Ldirect + Ld + Ls + emissive;
>```

>[!quote]- 通俗易懂的理解
>- 漫反射像“环境整体亮度”；镜面像“把环境里的形状映在物体上”。
>- 所以漫反射可以低频近似，镜面必须按粗糙度分层处理。

>[!example]- 面试回答简版模板
>`IBL就是用环境图做间接光。diffuse路径用irradiance近似半球积分，specular路径用预过滤环境图加BRDF LUT。最终把直接光、漫反射IBL、镜面IBL和自发光合成。`

## 题目：全局光照算法了解哪些？
>[!NOTE]- 标准准确的说法（每个算法单独讲）
>- **先把 GI 的目标说清楚**
>- 全局光照（Global Illumination, GI）要处理的是“光线多次反弹”。
>- 直接光只算“灯 -> 物体”；GI 要算“灯 -> 墙 -> 物体 -> 再反弹...”。
>- 因此 GI 会带来：颜色串色、暗部回光、间接阴影等真实感。
>- 
>- **一眼看懂：GI 算法其实是三大类**
>- 离线高质量：Path Tracing、Photon Mapping、Radiosity。
>- 实时近似：IBL、Light Probe、SSGI、RSM、LPV、VXGI、DDGI。
>- 实时光追：RTGI（通常配降噪）。
>- 
>- **1）Path Tracing（路径追踪）**
>- 是什么：从相机像素发射很多随机路径，模拟光线多次反弹，再做统计平均。
>- 怎么做：按 BRDF 随机采样方向，累计每次反弹能量，直到终止（或 Russian Roulette）。
>- 优点：物理一致性最好，几乎“万物可算”（软阴影、焦散、复杂间接光）。
>- 缺点：噪声大、收敛慢，实时里成本很高。
>- 适用：离线渲染、电影级画质；实时常作为“基准真值”或高端模式。
>- 面试一句话：`Path Tracing是通用且高真实度的蒙特卡洛GI，但需要大量样本与降噪。`
>- 
>- **2）Photon Mapping（光子映射）**
>- 是什么：先从光源发射“光子”存进空间数据结构，再从相机端查询附近光子估计间接光。
>- 怎么做：两阶段。
>- 阶段A：光源发射并记录光子落点（常见 kd-tree）。
>- 阶段B：着色点查询邻域光子做密度估计。
>- 优点：对焦散等“难算高亮现象”比较友好。
>- 缺点：参数敏感（搜索半径、光子数），可能有偏差和斑点伪影。
>- 适用：传统离线流程、焦散表现需求高的场景。
>- 面试一句话：`Photon Mapping用“先存光子再查询”的思路换取某些复杂间接光效果。`
>- 
>- **3）Radiosity（辐射度）**
>- 是什么：把场景表面离散成很多小面片，解面片之间的能量交换。
>- 怎么做：建立 form factor（形状因子）矩阵，迭代求解能量平衡。
>- 优点：对纯漫反射场景稳定，结果平滑。
>- 缺点：不擅长镜面/高频现象，几何复杂时矩阵开销大。
>- 适用：早期建筑可视化、纯漫反射主导场景。
>- 面试一句话：`Radiosity本质是面片级能量交换求解，强项是漫反射间接光。`
>- 
>- **4）IBL（Image Based Lighting）**
>- 是什么：用环境图近似来自四面八方的间接光。
>- 怎么做：diffuse 走 irradiance；specular 走 prefilter env + BRDF LUT。
>- 优点：成本低、观感提升大，是实时PBR标配。
>- 缺点：局部遮挡和动态互反射能力有限。
>- 适用：几乎所有实时项目。
>- 面试一句话：`IBL是实时GI的基础保底方案，性价比极高。`
>- 
>- **5）Light Probe（光照探针）**
>- 是什么：在场景放采样点，预存局部环境光信息，运行时插值使用。
>- 怎么做：常见用 SH 存低频漫反射；也可配 reflection probe 给镜面。
>- 优点：稳定、低成本、覆盖大场景。
>- 缺点：探针稀疏时会“糊”或过渡突变。
>- 适用：开放世界、室内外混合大场景的基础 GI。
>- 面试一句话：`Probe是“预采样+插值”的工程实用解。`
>- 
>- **6）SSGI（Screen Space GI）**
>- 是什么：在屏幕空间估计间接光（思路类似 SSR/SSAO 的屏幕空间近似）。
>- 怎么做：基于深度/法线/颜色在屏幕空间采样与追踪，再时域稳定。
>- 优点：不依赖离线烘焙，动态响应好，成本中等。
>- 缺点：屏幕外信息缺失，容易有闪烁/漏光。
>- 适用：需要动态 GI 感受、但预算不能上全 RT 的项目。
>- 面试一句话：`SSGI便宜且动态，但天生有“屏幕外看不到”的边界。`
>- 
>- **7）RSM（Reflective Shadow Maps）**
>- 是什么：在光源视角的 shadow map 里附加法线/反照率，估计一次反弹光。
>- 怎么做：用光源可见点作为“虚拟点光源”近似间接贡献。
>- 优点：实现思路直观，成本相对可控。
>- 缺点：通常只擅长一跳，覆盖和精度有限。
>- 适用：希望快速补“有一点间接光”效果的方案。
>- 面试一句话：`RSM是“借光源视角数据估一次反弹”的实时近似。`
>- 
>- **8）LPV（Light Propagation Volumes）**
>- 是什么：把光注入 3D 网格体积，再做若干步传播。
>- 怎么做：先注入，再在体积格子中迭代扩散。
>- 优点：有体积传播感，预算中等。
>- 缺点：容易漏光，细节会被平滑掉。
>- 适用：中等预算下追求“空间有光扩散感”的项目。
>- 面试一句话：`LPV是“在体积网格里传播光”的近似GI。`
>- 
>- **9）VXGI（Voxel GI）**
>- 是什么：把场景体素化后，在体素结构里做光传输与查询。
>- 怎么做：体素化场景 -> 注入/传播光 -> 着色时查询体素GI。
>- 优点：空间表达更完整，能处理较复杂遮挡与间接关系。
>- 缺点：显存和带宽压力较大，工程复杂度高。
>- 适用：高端实时项目、对动态GI要求较高场景。
>- 面试一句话：`VXGI用体素表示空间换取更完整GI，但代价是内存与性能。`
>- 
>- **10）DDGI（Dynamic Diffuse GI）**
>- 是什么：动态更新的 diffuse probe GI（可理解为“会刷新的探针GI”）。
>- 怎么做：在探针点做射线采样，更新辐照度与可见性，再插值到像素。
>- 优点：比静态 probe 更能响应动态变化，质量和成本平衡较好。
>- 缺点：探针布局、更新频率和泄漏控制需要精调。
>- 适用：需要“动态漫反射GI + 可控性能”的中高端实时项目。
>- 面试一句话：`DDGI是探针GI的动态升级版，核心是“可见性+时序更新”。`
>- 
>- **11）RTGI（Ray Traced GI）**
>- 是什么：用硬件/软件光追直接求间接光。
>- 怎么做：每像素发少量 GI 射线 + 时域积累 + 空间/时域降噪。
>- 优点：几何一致性强，动态场景质量高。
>- 缺点：性能昂贵，噪声控制与稳定化是核心难点。
>- 适用：高端平台、次世代项目。
>- 面试一句话：`RTGI质量高但成本高，工程上通常与降噪和分级策略绑定。`
>- 
>- **工程里怎么落地（最实用）**
>- 低端机：`IBL + AO + Probe`。
>- 中端机：`IBL + Probe/DDGI + SSGI`。
>- 高端机：`IBL + Probe/DDGI + RTGI（局部或全局）`。
>- 重点不是“选一个最强”，而是“按平台预算做组合”。

>[!tip]- 快速记忆表（你复习时最好用）
>| 算法 | 核心思路 | 质量 | 性能成本 | 典型短板 |
>|---|---|---|---|---|
>| Path Tracing | 蒙特卡洛多次反弹 | 很高 | 很高 | 噪声/收敛慢 |
>| Photon Mapping | 先存光子再查询 | 高 | 高 | 参数敏感 |
>| Radiosity | 面片能量交换 | 中-高（漫反射） | 高 | 不擅长镜面 |
>| IBL | 环境图近似间接光 | 中 | 低 | 局部互动弱 |
>| Probe | 预采样+插值 | 中 | 低 | 细节有限 |
>| SSGI | 屏幕空间估计GI | 中 | 中 | 屏幕外缺失 |
>| RSM | 光源图估一跳反弹 | 中 | 中 | 覆盖有限 |
>| LPV | 体积网格传播光 | 中 | 中 | 漏光/过平滑 |
>| VXGI | 体素化后查询传播 | 中-高 | 中-高 | 内存压力 |
>| DDGI | 动态探针GI | 中-高 | 中-高 | 调参复杂 |
>| RTGI | 实时光追间接光 | 高 | 高 | 降噪难 |

>[!quote]- 通俗易懂的理解
>- 这些算法不是“谁淘汰谁”的关系，而是不同预算下的工具箱。
>- 真正工程能力是：知道什么时候用哪把工具、怎么混合。

>[!example]- 面试回答简版模板
>`全局光照算法我会分三层：离线高质量（Path Tracing/Photon Mapping/Radiosity）、实时近似（IBL/Probe/SSGI/RSM/LPV/VXGI/DDGI）和实时光追（RTGI）。项目里通常按平台做混合方案，低端以IBL+Probe保底，高端再叠加RTGI并配降噪与时域稳定。`

## 题目：SSR的实现？Ray Marching是什么？
>[!NOTE]- 标准准确的说法（术语拆开讲）
>- `SSR`（Screen Space Reflection）是“屏幕空间反射”。
>- 它只使用当前帧已有的颜色和深度信息来做反射。
>- 
>- `Ray Marching` 可以翻成“沿射线逐步前进并检测”。
>- 具体就是：
>- 从当前像素出发，沿反射方向每次走一小步；
>- 每走一步都拿当前位置去和深度缓冲比较，看是否撞到表面。
>- 
>- **SSR 典型流程**
>- 1. 从 GBuffer 读 `N`（法线）、`V`（视线）、`depth`。
>- 2. 求反射方向 `R`。
>- 3. 在屏幕空间沿 `R` 做 ray marching。
>- 4. 命中则采样反射颜色。
>- 5. 未命中则 fallback 到 IBL/反射探针。
>- 6. 最后做时域稳定，减少闪烁。
>- 
>- **为什么会有空洞和断裂？**
>- 因为 SSR 只看得到屏幕里已经渲染出来的东西。
>- 屏幕外物体、被当前物体遮住的部分，它都不知道。

>[!tip]- 代码示例
>```cpp
>// 概念伪代码
>bool hit = MarchInScreenSpace(ray, depthBuffer, hitUV);
>float3 refl = hit ? ColorTex.Sample(samp, hitUV).rgb : FallbackIBL(R);
>```

>[!quote]- 通俗易懂的理解
>- SSR像“在当前照片里找镜子反射物”。
>- 照片外没有的信息，它不可能凭空算出来。

>[!example]- 面试回答简版模板
>`SSR是在屏幕空间沿反射方向做ray marching，通过深度比较找命中点。优点是便宜，缺点是只能反射屏幕内信息，因此常配合IBL或反射探针做补全。`

## 题目：光照探针/反射探针是什么？
>[!NOTE]- 标准准确的说法（先区分职责）
>- 探针可以理解成“提前放在场景里的采样点”。
>- 运行时不直接做昂贵积分，而是读取附近探针并插值。
>- 
>- **光照探针（Light Probe）**
>- 主要存低频间接光（常见是 SH 系数）。
>- 典型用于漫反射环境光。
>- 
>- **反射探针（Reflection Probe）**
>- 主要存方向相关的反射信息（常见是 cubemap）。
>- 典型用于镜面反射。
>- 
>- **运行时做什么**
>- 1. 找到当前位置附近探针。
>- 2. 根据距离/体积权重做插值。
>- 3. 漫反射走 light probe，镜面走 reflection probe。
>- 
>- **优缺点**
>- 优点：稳定、成本低、适合大场景。
>- 缺点：动态变化响应有限，探针稀疏会有过渡不自然。

>[!tip]- 代码示例
>```cpp
>// 概念伪代码
>ProbeSet probes = QueryNearbyProbes(worldPos);
>float3 Ld = EvalLightProbeSH(probes.lightProbeWeights, N);
>float3 Ls = EvalReflectionProbe(probes.reflProbeWeights, R, roughness);
>```

>[!quote]- 通俗易懂的理解
>- 探针像“气象站”：你在当前位置，参考附近几个站点的数据来估计天气。

>[!example]- 面试回答简版模板
>`Light Probe用于低频间接漫反射，Reflection Probe用于方向性反射。运行时按空间权重插值，优势是稳定高效，难点在探针布局和过渡处理。`

## 题目：RSM/LPV/VXGI的原理？
>[!NOTE]- 标准准确的说法（先讲共同目标）
>- 三者都在做同一件事：
>- 用可实时的数据结构去近似“光的传播”，避免直接做高成本全积分。
>- 
>- **RSM（Reflective Shadow Maps）**
>- 从光源视角渲染时，不只存深度，还存法线、反照率等。
>- 可近似估计一次反弹光（one-bounce）。
>- 特点：相对容易上手，但覆盖范围和精度有限。
>- 
>- **LPV（Light Propagation Volumes）**
>- 把光注入三维网格，让光在网格里传播几步。
>- 特点：有“光扩散”感觉，成本可控；但容易漏光、偏糊。
>- 
>- **VXGI（Voxel Global Illumination）**
>- 先把场景体素化，再在体素结构上做光照查询与传播。
>- 特点：空间表达更强；但内存和算力开销更高。
>- 
>- **怎么选**
>- 快速补一跳：RSM。
>- 预算中等且想要体积传播感：LPV。
>- 预算较高且追求更完整空间表达：VXGI。

>[!tip]- 通俗类比
>```text
>RSM  = 从灯的角度估一跳反光
>LPV  = 把光倒进3D网格里扩散
>VXGI = 先把世界体素化，再查间接光
>```

>[!quote]- 通俗易懂的理解
>- 三者区别主要不是目标，而是“用什么容器存光、怎么传播光”。

>[!example]- 面试回答简版模板
>`RSM/LPV/VXGI本质都是实时GI近似。RSM偏一次反弹估计，LPV用体积网格传播，VXGI用体素结构表达空间并查询间接光，三者是质量和成本的不同权衡。`

## 题目：AO(环境光遮蔽)的实现？
>[!NOTE]- 标准准确的说法（按“怎么实现”讲）
>- **先说结论**
>- AO（Ambient Occlusion）不是完整 GI，它只估计“环境光被周围几何遮住了多少”。
>- 最终通常输出一个遮蔽因子 `ao in [0,1]`，再去调制环境光。
>- 
>- 常见写法：
> $$
> L_{ambient} = ao \cdot L_{ambientBase}
> $$
>- 
>- AO 的理想形式可以写成半球可见性积分：
> $$
> AO(x)=1-\frac{1}{\pi}\int_{\Omega^+} V(x,\omega)(n\cdot\omega)\,d\omega
> $$
>- 其中 `V(x,\omega)` 代表该方向是否被遮挡（可见=1，遮挡=0）。
>- `SSAO/HBAO/GTAO/RTAO` 的差别，本质是“怎么近似求这个 `V`”。
>- 
>- **1）SSAO 怎么实现（最常见、最便宜）**
>- 输入：`depth + normal + noise texture`。
>- 实现步骤：
>- 1. 用深度重建当前像素视空间位置 `P`，读取法线 `N`。
>- 2. 准备半球采样核 `kernel[i]`（常见 16~32）。
>- 3. 用噪声纹理随机旋转采样核，减少条纹和 banding。
>- 4. 每个样本点 `S = P + TBN * kernel[i] * radius`。
>- 5. 把 `S` 投影回屏幕，取该处场景深度 `z_scene`。
>- 6. 比较 `z_scene` 与 `S.z` 判断是否遮挡，累计 `occlusion`。
>- 7. `ao = 1 - occlusion / sampleCount`。
>- 8. 对 AO 做法线/深度引导的双边滤波，减少噪声。
>- 优点：实现简单，性能友好，适合所有平台。
>- 缺点：屏幕外信息缺失，容易噪声/漏遮挡。
>- 
>- SSAO 常见离散形式：
> $$
> AO \approx 1-\frac{1}{N}\sum_{i=1}^{N} w_i\cdot \mathbf{1}(z_{scene}<z_{sample}-bias)
> $$
>- 
>- **2）HBAO / GTAO 怎么实现（SSAO 的质量升级）**
>- 共同点：不是只看“样本点是否挡住”，而是更重视方向几何关系，结果更稳定。
>- 
>- **HBAO（Horizon-Based AO）流程**
>- 1. 对每像素选若干屏幕方向（如 8 个方向）。
>- 2. 沿每个方向分步采样深度。
>- 3. 维护该方向“最高遮挡角”（地平线角）。
>- 4. 把多个方向的遮挡角贡献积分成 AO。
>- 5. 做保边滤波和时域稳定。
>- 特点：比 SSAO 更有体积感，噪点更少；成本更高。
>- 
>- **GTAO（Ground-Truth AO 近似）流程**
>- 1. 对每像素做方向切片（slice）采样。
>- 2. 在每个切片估计可见角区间（visibility arc）。
>- 3. 汇总所有切片得到 AO。
>- 4. 常同时输出 `bent normal`，给环境光采样方向做优化。
>- 5. 配合 temporal reprojection + spatial filter 做稳定化。
>- 特点：质量和稳定性通常优于 SSAO/HBAO，是实时项目常见高质量 AO 路线。
>- 
>- **3）RTAO 怎么实现（光追 AO）**
>- 输入：BVH/加速结构、法线、随机序列。
>- 实现步骤：
>- 1. 每像素从 `P,N` 出发，向半球发少量 AO 射线（常见 1~4 条/帧）。
>- 2. 设定最大检测半径 `radius`（只关心局部遮挡）。
>- 3. 射线在半径内命中几何体记“遮挡”，未命中记“可见”。
>- 4. 用命中率得到 `ao`（可加距离衰减）。
>- 5. 由于样本少会有噪点，必须做时域积累和降噪（如 A-trous / SVGF）。
>- 优点：几何一致性最好，不受屏幕外缺失影响。
>- 缺点：性能最贵，对降噪和时序稳定依赖很高。
>- 
>- RTAO 常见估计式：
> $$
> AO \approx 1-\frac{1}{N}\sum_{i=1}^{N}\mathbf{1}(\text{ray}_i\ \text{hit within}\ r)
> $$
>- 
>- **工程怎么选（非常实用）**
>- 低端预算：`SSAO + 双边滤波`。
>- 中高端默认：`GTAO + 时域稳定`。
>- 高端硬件：`RTAO + 降噪`（必要时和 GTAO 混合做分级）。
>- 
>- **避免踩坑**
>- AO 过强会“发黑发脏”，要限制强度和半径。
>- AO 不能替代 GI，最好只调制环境项或做受控混合。
>- 时域参数不当会拖影，运动区域要做历史降权。

>[!tip]- 代码示例
>```cpp
>// SSAO 概念伪代码（简化）
>float occlusion = 0.0;
>for (int i = 0; i < sampleCount; ++i) {
>    float3 S = P + TBN * kernel[i] * radius;
>    float2 uv = ProjectToScreen(S);
>    float zScene = DepthTex.Sample(samp, uv).r;
>    occlusion += (zScene < S.z - bias) ? 1.0 : 0.0;
>}
>float ao = 1.0 - occlusion / sampleCount;
>ao = BilateralFilter(ao, depth, normal);
>```

>[!quote]- 通俗易懂的理解
>- SSAO 是“屏幕上猜遮挡”，HBAO/GTAO 是“更聪明地猜遮挡”，RTAO 是“真的打射线看有没有挡住”。

>[!example]- 面试回答简版模板
>`AO核心是估计环境光可见性。SSAO便宜但屏幕外缺失，HBAO/GTAO通过方向几何近似提升稳定性与质量，RTAO用光追直接测遮挡最准确但成本最高。工程上通常按平台做分级并配合时域降噪。`

## 题目：球谐函数是什么？有什么好处？
>[!NOTE]- 标准准确的说法（详细版）
>- **先用一句人话定义**
>- 球谐函数（Spherical Harmonics, SH）就是：
>- 在“球面方向空间”里的一组基函数。
>- 它的角色，类似 1D 信号里的傅里叶基（正弦/余弦），只是这里对象变成了“方向”。
>- 
>- **为什么图形学会大量使用 SH？**
>- 因为很多光照信号是“方向相关但变化平滑”的低频信号（例如天空漫反射环境光）。
>- SH 能用很少系数压缩这种信号，运行时计算也很快。
>- 
>- **1）SH 的数学形式（你不用全背，但要会解释）**
>- 球面上的函数 `f(\omega)` 可用 SH 展开：
> $$
> f(\omega) \approx \sum_{l=0}^{L}\sum_{m=-l}^{l} c_{l,m}\,Y_{l,m}(\omega)
> $$
>- `Y_{l,m}`：SH 基函数。
>- `c_{l,m}`：对应基函数的系数。
>- `l` 是带（band，阶层），`m` 是该带内索引。
>- 
>- 系数由投影得到：
> $$
> c_{l,m}=\int_{S^2} f(\omega)Y_{l,m}(\omega)\,d\omega
> $$
>- 直觉上就是：`f` 在每个基函数方向模式上“有多像”。
>- 
>- **2）为什么经常说“9 个系数”？**
>- 如果取到 `L=2`（0/1/2 三个带），系数数量是：
> $$
> N=(L+1)^2=9
> $$
>- 这是实时渲染最经典折中：
>- `9` 个系数足够表达低频漫反射环境光。
>- 成本低，质量通常可接受。
>- 
>- RGB 三通道时常是 `9 x 3 = 27` 个浮点数。
>- 
>- **3）SH 在实时渲染里具体怎么用？**
>- 最常见：Light Probe 的 diffuse IBL。
>- 典型流程：
>- 1. 离线/预处理阶段：把环境光或探针采样值投影到 SH 系数。
>- 2. 运行时：给定法线方向 `N`，计算该方向的 SH 基函数值并线性组合。
>- 3. 得到该点的低频环境漫反射光照。
>- 
>- 运行时重建可写成：
> $$
> L_{diffuse}(N) \approx \sum_{i=1}^{9} c_i\,b_i(N)
> $$
>- 其中 `b_i(N)` 是将 `Y_{l,m}`按实现整理后的 9 个基函数。
>- 
>- **4）和卷积的关系（面试加分点）**
>- 漫反射本质是“环境光与余弦核”的卷积。
>- 在 SH 域里，卷积会变成系数乘法，计算非常高效。
>- 这就是 SH 适合 diffuse IBL 的核心原因之一。
>- 
>- **5）优点到底是什么？**
>- 存储小：低阶就能表达可用结果。
>- 计算快：运行时是少量点乘/线性组合。
>- 稳定：不会像高频方法那样容易闪烁。
>- 易插值：多个 probe 的 SH 系数可直接插值。
>- 
>- **6）边界和局限（这部分一定要会说）**
>- SH 擅长低频，不擅长高频。
>- 尖锐高光、镜面反射、硬阴影这类高频信号，用低阶 SH 会糊掉。
>- 所以工程上常见组合是：
>- diffuse 用 SH。
>- specular 用 prefiltered cubemap / BRDF LUT 等方法。
>- 
>- **7）阶数怎么选（工程经验）**
>- `L=2`（9系数）：实时最常见。
>- `L=3`（16系数）及以上：精度更高但成本增加。
>- 项目里通常先用 9 系数作为默认，再按质量档位扩展。
>- 
>- **8）你可以这样理解 SH**
>- SH 不是“把所有光照都算准”的万能工具。
>- 它是一个“低频压缩与快速重建工具”，特别适合 GI 里的漫反射底座。

>[!tip]- 代码示例
>```cpp
>// 概念伪代码：用9个SH基重建漫反射环境光
>float3 EvalDiffuseSH(float3 N, float3 sh[9])
>{
>    float b[9] = {
>        0.282095f,
>        0.488603f * N.y,
>        0.488603f * N.z,
>        0.488603f * N.x,
>        1.092548f * N.x * N.y,
>        1.092548f * N.y * N.z,
>        0.315392f * (3.0f * N.z * N.z - 1.0f),
>        1.092548f * N.x * N.z,
>        0.546274f * (N.x * N.x - N.y * N.y)
>    };
>
>    float3 L = float3(0, 0, 0);
>    for (int i = 0; i < 9; ++i) L += sh[i] * b[i];
>    return max(L, 0.0);
>}
>```
>
>```text
>工程常见分工：
>Diffuse IBL  -> SH（低频、高效）
>Specular IBL -> Prefiltered Env + BRDF LUT（高频）
>```

>[!quote]- 通俗易懂的理解
>- SH像“用少量、固定的方向波形去拟合天空光分布”。
>- 它不追求镜子级细节，但能非常省地提供稳定的漫反射环境光。

>[!example]- 面试回答简版模板
>`球谐函数是球面信号的基函数展开，适合表示低频方向光照。实时渲染中常用低阶SH（如9系数）存储Light Probe的漫反射环境光，运行时按法线快速重建。它的优势是压缩率高、计算便宜、稳定性好；局限是难表达高频镜面细节，所以通常和预过滤环境图等方法配合使用。`

## 题目：Lumen的了解？
>[!NOTE]- 标准准确的说法（先摆正定位）
>- Lumen 不是单一算法，而是 Unreal 中一套“混合式实时GI与反射系统”。
>- 目标是在动态场景里减少传统烘焙依赖，同时维持可用性能。
>- 
>- **它为什么是混合式**
>- 单一路线各有短板：
>- 只用屏幕空间不完整。
>- 只用硬件光追开销大。
>- 所以 Lumen 会组合多信息源，再统一稳定输出。
>- 
>- **可记忆的结构（概念级）**
>- 屏幕空间信息：便宜，先拿能拿到的。
>- 场景近似/软件追踪：补屏幕外信息。
>- 硬件 RT：在高档位进一步提高准确度。
>- 时域积累与去噪：把结果稳定下来。
>- 
>- **工程现实**
>- 效果和性能会随硬件、场景复杂度、质量档位显著变化。
>- 实际项目常做平台分级，不会所有子功能全开。

>[!tip]- 一句话结构
>```text
>Lumen ~= Screen-space + Software tracing + (Optional) Hardware RT + Temporal accumulation + Denoise
>```

>[!quote]- 通俗易懂的理解
>- Lumen更像“调度系统”，不是某个单独公式。

>[!example]- 面试回答简版模板
>`Lumen是实时GI与反射的混合框架，不依赖单一算法。它结合屏幕空间、软件/硬件追踪和时域稳定化，在动态场景里做画质与性能平衡。`

## 题目：SVGF了解吗？
>[!NOTE]- 标准准确的说法（先讲它解决什么）
>- SVGF（Spatiotemporal Variance-Guided Filtering）是“时空联合降噪”。
>- 主要用于实时光追、SSGI、SSR这类低采样高噪声信号。
>- 
>- **核心想法**
>- 时间上：借历史帧，先把随机噪声平均掉。
>- 空间上：根据方差决定滤波强度。
>- 噪声大就多滤，细节区就少滤。
>- 
>- **典型流程**
>- 1. 重投影历史帧（用运动矢量把历史像素对齐到当前帧）。
>- 2. 时间融合（temporal accumulation）。
>- 3. 估计方差（variance estimation）。
>- 4. 做多尺度空间滤波（常见 A-trous）。
>- 
>- **常见问题与对策**
>- 拖影（ghosting）：历史信息不可靠时要降权或丢弃。
>- 细节变糊：滤波核不能无脑放大，要用法线/深度引导。
>- 闪烁：需要稳定的重投影和合理 clamp。

>[!tip]- 流程示意
>```mermaid
>flowchart TD
>  A[当前帧噪声信号] --> B[重投影历史]
>  B --> C[时间融合]
>  C --> D[方差估计]
>  D --> E[A-trous多尺度滤波]
>  E --> F[稳定输出]
>```
>
>```cpp
>// 概念伪代码
>History h = ReprojectHistory(curr, motionVec);
>Signal t = TemporalAccumulate(curr, h);
>Variance v = EstimateVariance(t);
>Signal o = AtrousFilter(t, v, normal, depth);
>```

>[!quote]- 通俗易懂的理解
>- SVGF 像“先把前几帧拿来投票，再按噪声大小决定哪里多磨皮、哪里保细节”。

>[!example]- 面试回答简版模板
>`SVGF是时空联合降噪框架：先重投影和时间积累，再用方差引导多尺度空间滤波。它能在低采样下显著降噪，但需要控制拖影和细节损失。`

## 一页速记（考前30秒）
>[!NOTE]- 你最少要记住的几句话
>- IBL 分两路：`diffuse -> irradiance`，`specular -> prefilter + BRDF LUT`。
>- SSR 是“屏幕内反射”，靠 ray marching 找命中，屏幕外要 fallback。
>- Probe 是“预采样+插值”的低成本 GI 近似。
>- RSM/LPV/VXGI 是不同数据结构下的实时光传播近似。
>- AO 只做遮蔽增强，不等于完整 GI。
>- SH 适合低频漫反射，不适合尖锐镜面。
>- Lumen 是混合框架，不是单算法。
>- SVGF 是“历史帧 + 方差引导滤波”的降噪流程。
